    Mathematical Foundations of Machine Learning

    Chapter 42: Multi-Head Attention Layouts, Representation Normalization (Batch vs. Layer), and the Unified Transformer Block

    42.1 Multi-Head Attention: Parallel Representation Subspaces

    In Chapter 41, we unrolled the global matrix equation for Scaled Dot-Product Attention, proving how direct pairwise coordinate cross-referencing eliminates the sequential bottleneck of recurrent architectures. However, restricting a network to a single attention pass introduces a severe representational constraint: a single attention head can only extract one structural relationship pattern across the sequence at a time.

    If an engineering task requires tracking diverse, overlapping data relationships simultaneously—such as matching an adjective to a noun while simultaneously tracking long-range subject-verb agreements—a single attention head collapses these distinct channels into a single, blurry average.

    To resolve this limitation, we introduce Multi-Head Attention (MHA). Instead of optimizing one massive attention filter, we deploy a bank of \(M\) independent attention heads completely in parallel.

    1. Head Projection and Extraction

    For each individual head index \(j\) (where \(1 \leq j \leq M\)), we assign a unique, independent set of learnable parameter matrices: \(\mathbf{W}_Q^{(j)}, \mathbf{W}_K^{(j)}, \mathbf{W}_V^{(j)}\). We project our global data matrix \(\mathbf{X}\) into \(M\) distinct head tracking spaces, computing a parallel bank of attention context matrices \(\mathbf{Z}_1, \mathbf{Z}_2, \dots, \mathbf{Z}_M\):

    \[\mathbf{Z}_j = \text{Attention}\left(\mathbf{X}\mathbf{W}_Q^{(j)}, \,\, \mathbf{X}\mathbf{W}_K^{(j)}, \,\, \mathbf{X}\mathbf{W}_V^{(j)}\right) \in \mathbb{R}^{T \times D_V}\]

    2. Tensor Synthesis via Channel Merging

    By running this parallel extraction loop, we obtain \(M\) independent matrices, each capturing a unique aspect of token interaction geometry. To unify these isolated channels back into a single cohesive stream, we flatly concatenate all \(M\) heads side-by-side along their column dimension, forming an expanded tensor volume:

    \[\mathbf{Z}_{\text{concat}} = \Big[ \mathbf{Z}_1, \,\, \mathbf{Z}_2, \,\, \dots, \,\, \mathbf{Z}_M \Big] \in \mathbb{R}^{T \times (M \cdot D_V)}\]

    3. Subspace Compression

    This concatenated tensor volume is too wide to pass directly into subsequent deep layers. To compress it back into our native hidden dimension width (\(D_V\)), we apply a final, learnable linear synthesis matrix denoted as \(\mathbf{W}_Z\):

    \[\text{MultiHead}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \mathbf{Z}_{\text{concat}} \mathbf{W}_Z\]

    Precise Dimensional Balancing

    To ensure structural consistency across the layer, we configure the parameter dimensions of our output projection matrix \(\mathbf{W}_Z\) to precisely map the concatenated vectors back to our target feature dimensions:

    \[\mathbf{Z}_{\text{concat}} \in \mathbb{R}^{T \times (M \cdot D_V)} \quad \times \quad \mathbf{W}_Z \in \mathbb{R}^{(M \cdot D_V) \times D_V} \quad \implies \quad \mathbf{Z}_{\text{MHA}} \in \mathbb{R}^{T \times D_V}\]

    The professor highlights the direct structural analogy between this multi-head architecture and the parallel filter volumes of a Convolutional Neural Network (CNN). In a CNN, we do not limit a layer to a single sliding kernel; we deploy dozens of independent kernels to capture diverse spatial features like edges, textures, and color gradients simultaneously. Multi-Head Attention applies this exact same probabilistic strategy to sets of vectors, allowing the model to analyze data interactions from multiple distinct geometric perspectives concurrently.

    42.2 The Epistemological Leap: Shifting from Parameter to Representation Regularization

    Up to this point in the course, our mathematical strategy for controlling model variance and fighting overfitting has focused exclusively on Parameter Regularization.

    \[\min_{\vec{\theta}} \hat{R}(\vec{\theta}) + \Omega(\vec{\theta})\]

    Under this classical empirical risk framework, we penalize the weight coefficients directly—using an \(L_2\) norm penalty (\(\lambda \|\vec{w}\|_2^2\)) to enforce a smooth Gaussian prior, or an \(L_1\) norm penalty to force sparse coordinate choices.

    However, let us analyze a composite neural network function from an alternate structural lens. We unroll a multi-layer composition where the final hypothesis is a function of both its weights (\(\vec{\theta}\)) and its intermediate hidden activations (\(\vec{z}\)):

    \[h_{\vec{\theta}}(\vec{x}) = f_{\mathbf{W}_2}\left( \vec{z} \right) \quad \text{where} \quad \vec{z} = \sigma\left(\mathbf{W}_1 \vec{x}\right)\]

    Because the final prediction depends intimately on both components, we can regularize the model by placing structural constraints directly on the intermediate hidden representation space \(\vec{z}\). We call this strategy Activity Regularization or Representation Normalization.

    Instead of policing the weights, we actively constrain the hidden activation coordinates generated during the forward pass. This forces them to conform to a stable distribution envelope, preventing activation magnitudes from exploding or drifting across layers.

    42.3 Representation Normalization Mechanics: Batch vs. Layer

    To implement representation regularization, we replace a raw hidden activation vector \(\vec{z}\) with a normalized coordinate variant. This transformation forces the hidden distribution to maintain a stable statistical profile characterized by a mean of \(0.0\) and a variance of \(1.0\):

    \[\hat{\vec{z}} = \frac{\vec{z} - \mu_z}{\sqrt{\sigma_z^2 + \epsilon}}\]
    \[\vec{z}_{\text{norm}} = \vec{\gamma} \odot \hat{\vec{z}} + \vec{\beta}\]

    The Learnable Safety Valve (\(\vec{\gamma}, \vec{\beta}\)):

    The vectors \(\vec{\gamma}\) (scale) and \(\vec{\beta}_0\) (shift) are learnable parameters updated via gradient descent during our ERM optimization loop. These parameters function as an architectural safety valve.

    If the optimization engine discovers that a layer performs better without normalization constraints, it can tune \(\vec{\gamma}\) to exactly match the incoming standard deviation and \(\vec{\beta}_0\) to match the raw mean. This configuration allows the network to completely undo the normalization step and pass the raw activations forward unfiltered, ensuring the constraint never damages model expressivity.

      [ Raw Activations Matrix Z ] ──────────> [ Compute Mean & Variance Over Chosen Axis ]
               │                                                    │
               ▼                                                    ▼
  [ Center and Scale Data ] ───────────> [ Scale and Shift by Learnable Weights γ and β ]
                                                                    │
                                                                    ▼
                                                       [ Normalized Activations Layer ]

    The difference between the two most dominant normalization strategies in deep learning depends entirely on which structural axis of our data matrix the mean (\(\mu\)) and variance (\(\sigma^2\)) are calculated over.

    1. Batch Normalization (BatchNorm)

    In BatchNorm, the statistical parameters are calculated across the batch dimension for a single independent feature coordinate. We average the activations of multiple distinct data samples processed within a single SGD training mini-batch:

    \[\mu_{\text{Batch}} = \frac{1}{B} \sum_{i=1}^{B} z_{i, \text{channel}}\]
    • Application Fit: BatchNorm is highly effective in image processing networks and standard CNN architectures.
    • The Inference Bottleneck: Because BatchNorm depends on computing statistics across a mini-batch of multiple samples, it creates a structural challenge during production deployment. When a single user submits an isolated sample during inference, a batch no longer exists. To bypass this, engineering frameworks must calculate and store a running average of training statistics beforehand, then freeze and swap these values in during inference.

    2. Layer Normalization (LayerNorm)

    In LayerNorm, the statistical parameters are calculated across the feature dimensions of a single, isolated data sample. We average all coordinate values within that layer for one individual token position, completely ignoring neighboring batch samples:

    \[\mu_{\text{Layer}} = \frac{1}{D_{\text{layer}}} \sum_{j=1}^{D_{\text{layer}}} z_{\text{feature}_j}\]
    • Application Fit: LayerNorm is the foundational normalization standard for Transformers and sequential architectures. Because its statistics are calculated entirely within the boundary of a single data sample, it completely eliminates the inference mini-batch bottleneck, allowing isolated text or video strings to normalize consistently during live production deployment.

    Eliminating Internal Covariant Shift

    We deploy these normalization steps to prevent a classic deep learning failure mode known as Internal Covariant Shift. As deep gradient descent updates early weight parameters (\(\mathbf{W}_1\)), the distribution of hidden features entering subsequent layers shifts constantly from epoch to epoch.

    This shifting forces deeper layers (\(\mathbf{W}_2\)) to continuously chase a moving target, slowing down convergence. Centering and scaling the activations ensures that every hidden layer receives input data bounded within a stable, stationary distribution envelope, accelerating optimization and smoothing the global loss landscape.

    42.4 The Anatomy of a Unified Transformer Block

    We now combine our structural building blocks—Gated Attention, Multi-Head Parallelism, Residual Identity Highways, Layer Normalization, and Non-linear Multi-Layer Perceptrons—into a single, production-ready Transformer Block Architecture.

    A standard Transformer layer consists of a highly structured, two-stage processing lattice:

    Stage 1: The Context Routing Lattice

    An incoming token sequence matrix \(\mathbf{X} \in \mathbb{R}^{T \times D}\) enters the block and splits immediately into two parallel paths: a non-linear Multi-Head Attention block and a clean, spatial identity skip connection. The outputs of these paths are combined and passed through a Layer Normalization wrapper to construct our first intermediate state matrix \(\mathbf{X}_1\):

    \[\mathbf{X}_1 = \text{LayerNorm}\Big( \mathbf{X} + \text{MultiHead}(\mathbf{X}) \Big) \in \mathbb{R}^{T \times D}\]

    Stage 2: The Non-Linear Expressive Lattice

    The intermediate matrix \(\mathbf{X}_1\) splits along a second parallel track. It passes through a dense, token-wise Multi-Layer Perceptron (MLP) block to introduce non-linear expressive capacity, while a parallel skip connection preserves the raw state. The results are merged and normalized to yield the final output tensor \(\mathbf{X}_2\):

    \[\mathbf{X}_2 = \text{LayerNorm}\Big( \mathbf{X}_1 + \text{MLP}(\mathbf{X}_1) \Big) \in \mathbb{R}^{T \times D}\]

    The Mechanics of Token Isolation: Notice how the workload shifts between these two stages. The Multi-Head Attention stage is the only component where tokens are allowed to communicate across rows to share context. Once \(\mathbf{X}_1\) is generated, the subsequent MLP stage operates as a strict Position-Wise Feed-Forward Network. It processes each token row vector independently and in parallel using a shared set of weights. The MLP introduces the non-linear transformations required to satisfy the Universal Approximation Theorem, while the attention stage routes features across the timeline.

    42.5 Advanced Architectural Frontiers: State Space Models (Mamba)

    The professor concludes the sequence modeling continuum by analyzing the modern trade-offs that drive cutting-edge machine learning research.

    While the Transformer architecture successfully eliminated the vanishing gradient problem of RNNs, it introduced an expensive computational bottleneck: Quadratic Scaling Overhead (\(\mathcal{O}(T^2)\)). Because the attention mechanism cross-references every token against every other token, calculating the dot-product adjacency matrix requires computing \(T \times T\) inner products. As the context window threshold expands to handle large datasets, the compute and memory costs scale quadratically, making long-sequence inference highly expensive.

      [ Recurrent RNN Engine ]                 [ Attention Transformer Engine ]
  - Compute Cost: O(T) Linear              - Compute Cost: O(T²) Quadratic
  - Memory Footprint: Constant O(1)        - Memory Footprint: Expanding O(T²) Cache
  - Limit: Gradient Decay Risk             - Limit: Heavy Computational Overhead

    To break this quadratic bottleneck without re-introducing the vanishing gradient failures of vanilla networks, modern hardware research has advanced to a new architectural frontier: State Space Models (SSMs), exemplified by systems like Mamba.

    An SSM brings recurrence back into the architecture, modeling data streams as a continuous linear dynamical system. However, instead of using rigid transition matrices, an SSM deploys Selection Mechanisms—dynamic, data-dependent parameters that update continuously at each time step. By combining this selective recurrence with highly parallelized prefix-sum hardware algorithms, State Space Models achieve linear computational scaling (\(\mathcal{O}(T)\)) and a constant memory footprint (\(\mathcal{O}(1)\)) during inference, while matching the long-context expressive capacity of modern Transformers.
