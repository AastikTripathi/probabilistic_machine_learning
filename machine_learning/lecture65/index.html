    Chapter 68: Latent Variable Generative Models and Variational Autoencoders

    68.1 A New Family: Models That Think in Two Spaces

    The GAN framework of Chapter 67 is a decoder-only generative model. It defines a map \(G_\theta : \mathcal{Z} \to \mathcal{X}\) from a latent noise space to the data space, and at inference time we simply draw \(\mathbf{z} \sim p_Z\) and apply \(G_\theta\). The model has no mechanism for asking the inverse question: given an observed data point \(\mathbf{x}\), what is the corresponding latent representation? The discriminator \(D_\phi\), which did have access to \(\mathbf{x}\), is discarded after training.

    This chapter introduces a richer family: latent variable generative models. These models are built around the idea that data \(\mathbf{x} \in \mathcal{X}\) is generated by a two-step process:

    1. Draw a latent variable \(\mathbf{z} \in \mathcal{Z}\) from some prior \(p(\mathbf{z})\).
    2. Draw \(\mathbf{x}\) from a conditional distribution \(p_\theta(\mathbf{x} \mid \mathbf{z})\).

    The joint distribution is \(p_\theta(\mathbf{x}, \mathbf{z}) = p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})\), and the marginal of interest is:

    \[p_\theta(\mathbf{x}) = \int_{\mathcal{Z}} p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z}) \, d\mathbf{z} \tag{68.1}\]

    This is the same latent variable model structure introduced in the EM and mixture model chapters of this course. The new element is that we now use it explicitly for generation: by learning a good \(p_\theta(\mathbf{x} \mid \mathbf{z})\), we gain a decoder that maps latent samples to data samples. Because the latent space is also equipped with a prior \(p(\mathbf{z})\), the model can run in both directions — encoding data into latents and decoding latents into data.

    Definition 68.1 (Encoder and Decoder). In a latent variable model: - The encoder is the posterior \(p_\theta(\mathbf{z} \mid \mathbf{x})\): given a data point \(\mathbf{x}\), it returns the distribution over latent codes \(\mathbf{z}\) that could have generated \(\mathbf{x}\). It is the feature extractor or embedding function. - The decoder is the conditional \(p_\theta(\mathbf{x} \mid \mathbf{z})\): given a latent code \(\mathbf{z}\), it returns the distribution over data points \(\mathbf{x}\). It is the sampler or generator.

    This dual nature is what distinguishes latent variable generative models from GANs:

    Property GAN Latent Variable Model
    Generates new samples Yes (\(G_\theta(\mathbf{z})\)) Yes (decoder)
    Extracts embeddings No Yes (encoder)
    Has tractable \(p_\theta(\mathbf{x})\) No Sometimes
    Architecture type Decoder-only Encoder-decoder

    The Variational Autoencoder (VAE) is the canonical example of a latent variable generative model trained with neural networks. Before deriving it, we revisit a simpler and already-familiar model that is, in fact, also a latent variable generative model: the Gaussian Mixture Model.

    68.2 The Gaussian Mixture Model as a Generative Model

    68.2.1 Review of the GMM Structure

    We have studied the GMM extensively as a clustering and density estimation method. Recall the model:

    \[p_\theta(\mathbf{x}, z) = p_\theta(\mathbf{x} \mid z) \, p_\theta(z) = \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_z, \boldsymbol{\Sigma}_z) \cdot \pi_z \tag{68.2}\]

    where \(z \in \{1, 2, \ldots, M\}\) is a discrete latent variable (the component index), \(\pi_z = p(z)\) is the mixing proportion for component \(z\), and \(\boldsymbol{\mu}_z, \boldsymbol{\Sigma}_z\) are the mean and covariance of the \(z\)-th Gaussian. The marginal is:

    \[p_\theta(\mathbf{x}) = \sum_{z=1}^M \pi_z \, \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_z, \boldsymbol{\Sigma}_z) \tag{68.3}\]

    The parameter set is \(\theta^* = \{\pi_z^*, \boldsymbol{\mu}_z^*, \boldsymbol{\Sigma}_z^*\}_{z=1}^M\), estimated via the EM algorithm.

    68.2.2 GMM Generative Inference: A Two-Step Sampler

    Once \(\theta^*\) is known, the GMM is a fully operational generative model. Sampling from \(p_{\theta^*}(\mathbf{x})\) decomposes into exactly the two steps prescribed by the latent variable framework:

    Step 1: Sample the latent variable. Draw \(\hat{z} \sim p_{\theta^*}(z) = \text{Categorical}(\pi_1^*, \pi_2^*, \ldots, \pi_M^*)\).

    Step 2: Sample from the conditional. Given \(\hat{z}\), draw \(\tilde{\mathbf{x}} \sim p_{\theta^*}(\mathbf{x} \mid z = \hat{z}) = \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_{\hat{z}}^*, \boldsymbol{\Sigma}_{\hat{z}}^*)\).

    The output \(\tilde{\mathbf{x}}\) is a sample from \(p_{\theta^*}(\mathbf{x})\), the fitted mixture density.

    Step 1 in detail: Sampling from a discrete distribution. We assumed we know how to sample from \(\text{Uniform}[0,1]\). How do we convert that into a sample from \(\text{Categorical}(\pi_1, \ldots, \pi_M)\)?

    This is precisely the Inverse CDF method applied to a discrete distribution. Define the cumulative probabilities:

    \[F(k) = \sum_{j=1}^k \pi_j, \quad k = 1, \ldots, M, \quad F(0) = 0\]

    Draw \(u \sim \text{Uniform}[0,1]\) and return \(\hat{z} = k\) where \(k\) is the unique index satisfying \(F(k-1) < u \leq F(k)\).

    Why this works. The probability of returning \(\hat{z} = k\) is:

    \[\mathbb{P}(\hat{z} = k) = \mathbb{P}(F(k-1) < U \leq F(k)) = F(k) - F(k-1) = \pi_k\]

    which matches the target distribution exactly. \(\blacksquare\)

    Worked Example 68.1: Discrete Sampling. Suppose \(M = 3\) with \(\pi_1 = 0.5\), \(\pi_2 = 0.3\), \(\pi_3 = 0.2\). The CDF breakpoints are \(F(1) = 0.5\), \(F(2) = 0.8\), \(F(3) = 1.0\).

    If \(u = 0.37\): \(F(0) = 0 < 0.37 \leq 0.5 = F(1)\), so \(\hat{z} = 1\). If \(u = 0.72\): \(F(1) = 0.5 < 0.72 \leq 0.8 = F(2)\), so \(\hat{z} = 2\). If \(u = 0.91\): \(F(2) = 0.8 < 0.91 \leq 1.0 = F(3)\), so \(\hat{z} = 3\).

    Over many draws, component 1 is selected 50\% of the time, component 2 is selected 30\%, and component 3 is selected 20\% — matching \((\pi_1, \pi_2, \pi_3)\).

    The brute-force implementation of this (linear scan) is \(O(M)\). By precomputing the CDF and using binary search, it is reducible to \(O(\log M)\) — a useful observation for large \(M\).

    Step 2 in detail. Once \(\hat{z} = k\) is selected, we need \(\tilde{\mathbf{x}} \sim \mathcal{N}(\boldsymbol{\mu}_k^*, \boldsymbol{\Sigma}_k^*)\). If \(\boldsymbol{\Sigma}_k^* = \mathbf{L}_k \mathbf{L}_k^\top\) (Cholesky factorization), then:

    \[\tilde{\mathbf{x}} = \boldsymbol{\mu}_k^* + \mathbf{L}_k \boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I}_d)\]

    We already know how to draw \(\boldsymbol{\epsilon}\) (via Box-Muller or its generalization). So the full GMM sampler requires only operations we already have primitives for.

    Example 68.2: GMM on MNIST. Fit a \(M\)-component GMM (via EM) to the \(d = 784\)-dimensional MNIST digit images with no labels. Each pixel is one dimension; the GMM models the joint distribution over all 784 pixels. Post-fitting, to generate a new image:

    1. Sample a component \(\hat{z} \in \{1, \ldots, M\}\) using the inverse-CDF method on \((\hat{\pi}_1, \ldots, \hat{\pi}_M)\).
    2. Sample \(\tilde{\mathbf{x}} \sim \mathcal{N}(\hat{\boldsymbol{\mu}}_{\hat{z}}, \hat{\boldsymbol{\Sigma}}_{\hat{z}})\), a 784-dimensional vector.
    3. Reshape to \(28 \times 28\) and display.

    Each component \(\hat{z}\) loosely corresponds to a cluster of visually similar digits (perhaps not perfectly separated by class, since EM has no access to labels). The generated image is a novel pixel vector that the fitted distribution assigns nonzero probability to.

    68.2.3 Why the GMM Limits Us

    The GMM illustrates the latent variable generative model principle beautifully but has two fundamental limitations that motivate the VAE:

    Expressivity. The conditional \(p_\theta(\mathbf{x} \mid z) = \mathcal{N}(\mathbf{x} \mid \boldsymbol{\mu}_z, \boldsymbol{\Sigma}_z)\) is a Gaussian — a unimodal, symmetric distribution in the data space. Real data distributions (images, audio) are vastly more complex than any mixture of Gaussians in the pixel space. To capture fine structure, we would need \(M \to \infty\) components, which is computationally intractable.

    Tractability of the posterior. In the GMM, the posterior \(p_\theta(z \mid \mathbf{x})\) is computable via Bayes' rule because both \(p_\theta(\mathbf{x} \mid z)\) and \(p_\theta(z)\) are in closed form. The EM algorithm exploits this. For more expressive conditionals (e.g., \(p_\theta(\mathbf{x} \mid \mathbf{z})\) given by a deep neural network), the posterior \(p_\theta(\mathbf{z} \mid \mathbf{x}) = p_\theta(\mathbf{x} \mid \mathbf{z}) p(\mathbf{z}) / p_\theta(\mathbf{x})\) involves the intractable integral \(p_\theta(\mathbf{x}) = \int p_\theta(\mathbf{x} \mid \mathbf{z}) p(\mathbf{z}) d\mathbf{z}\) in the denominator. The encoder becomes unavailable.

    The Variational Autoencoder is precisely the solution to these two problems: it uses neural networks for both the encoder and decoder (solving the expressivity problem) and introduces variational inference to handle the intractable posterior (solving the tractability problem).

    68.3 Encoder-Decoder Taxonomy

    Before diving into the VAE, it is useful to situate it within the broader taxonomy of models we have seen in this course.

    Encoder-only models learn a mapping \(\mathcal{X} \to \mathcal{Z}\) from the data space to a latent space. They extract representations but cannot generate new data. Examples: SimCLR and other self-supervised contrastive learning methods, which learn feature extractors but discard the generative structure.

    Decoder-only models learn a mapping \(\mathcal{Z} \to \mathcal{X}\) from a latent space to the data space. They generate new data but cannot extract structured embeddings from existing data. Examples: GANs (Chapter 67), autoregressive language models (see §68.8).

    Encoder-decoder models maintain both directions simultaneously: an encoder \(\mathcal{X} \to \mathcal{Z}\) and a decoder \(\mathcal{Z} \to \mathcal{X}\). They support both generation and embedding extraction. Examples: VAEs (this chapter), VQ-VAEs, diffusion models in latent space.

    The mathematical content of each direction is:

    \[\text{Encoder:} \quad p_\theta(\mathbf{z} \mid \mathbf{x}) \quad \text{(posterior of latent given data)}$$ $$\text{Decoder:} \quad p_\theta(\mathbf{x} \mid \mathbf{z}) \quad \text{(conditional of data given latent)}\]

    The joint distribution \(p_\theta(\mathbf{x}, \mathbf{z}) = p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})\) gives the model; the marginal \(p_\theta(\mathbf{x}) = \int p_\theta(\mathbf{x} \mid \mathbf{z}) p(\mathbf{z}) d\mathbf{z}\) is what we want to match to the data distribution.

    68.4 Variational Autoencoders: Problem Setup

    68.4.1 The Model

    A Variational Autoencoder assumes the following generative process for data \(\mathbf{x} \in \mathbb{R}^d\):

    \[\mathbf{z} \sim p(\mathbf{z}) = \mathcal{N}(\mathbf{0}, \mathbf{I}_k) \tag{68.4}$$ $$\mathbf{x} \mid \mathbf{z} \sim p_\theta(\mathbf{x} \mid \mathbf{z}) \tag{68.5}\]

    where \(\theta\) parameterizes the decoder neural network \(f_\theta : \mathbb{R}^k \to \mathcal{X}\) (more precisely, to the parameter space of a distribution over \(\mathcal{X}\)). Common choices:

    • For continuous data: \(p_\theta(\mathbf{x} \mid \mathbf{z}) = \mathcal{N}(\mathbf{x} \mid f_\theta(\mathbf{z}), \sigma^2 \mathbf{I})\) — the decoder outputs the mean of a Gaussian.
    • For binary data: \(p_\theta(\mathbf{x} \mid \mathbf{z}) = \prod_{j=1}^d \text{Bernoulli}(x_j \mid \sigma(f_\theta(\mathbf{z}))_j)\) — the decoder outputs Bernoulli probabilities per pixel.

    The prior \(p(\mathbf{z}) = \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\) is fixed (not learned). The marginal:

    \[p_\theta(\mathbf{x}) = \int_{\mathbb{R}^k} p_\theta(\mathbf{x} \mid \mathbf{z}) \, \mathcal{N}(\mathbf{z} \mid \mathbf{0}, \mathbf{I}_k) \, d\mathbf{z} \tag{68.6}\]

    is intractable for deep network decoders.

    68.4.2 The Training Objective: Maximum Likelihood

    We would like to maximize the log-likelihood of the data:

    \[\max_\theta \sum_{i=1}^n \ln p_\theta(\mathbf{x}^{(i)}) = \max_\theta \sum_{i=1}^n \ln \int p_\theta(\mathbf{x}^{(i)} \mid \mathbf{z}) \, p(\mathbf{z}) \, d\mathbf{z} \tag{68.7}\]

    Both the integral itself and its gradient with respect to \(\theta\) are intractable. Direct naive Monte Carlo estimation of the gradient is possible but has extremely high variance — the samples from \(p(\mathbf{z})\) that contribute to \(p_\theta(\mathbf{x}^{(i)} \mid \mathbf{z})\) being large are rare for complex data. We need a smarter approach.

    68.4.3 Why the EM Approach Fails Here

    Recall the EM algorithm: it circumvents the intractable integral by alternating between (E) computing \(p_\theta(\mathbf{z} \mid \mathbf{x})\) and (M) maximizing the expected complete-data log-likelihood. The E-step requires computing the posterior:

    \[p_\theta(\mathbf{z} \mid \mathbf{x}) = \frac{p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})}{p_\theta(\mathbf{x})} \tag{68.8}\]

    For the GMM, \(p_\theta(\mathbf{x} \mid \mathbf{z})\) is Gaussian, so the numerator and denominator are both expressible in closed form, and (68.8) is tractable (it is just a softmax over mixture weights). For a deep neural network decoder, \(p_\theta(\mathbf{x} \mid \mathbf{z}) = \mathcal{N}(\mathbf{x} \mid f_\theta(\mathbf{z}), \sigma^2 \mathbf{I})\) has no closed-form normalizer as a function of \(\mathbf{z}\), so \(p_\theta(\mathbf{x}) = \int p_\theta(\mathbf{x} \mid \mathbf{z}) p(\mathbf{z}) d\mathbf{z}\) is intractable. The E-step breaks down.

    The VAE's solution: replace the intractable true posterior \(p_\theta(\mathbf{z} \mid \mathbf{x})\) with a learned approximate posterior (encoder) \(q_\phi(\mathbf{z} \mid \mathbf{x})\), and derive a tractable surrogate objective.

    68.5 The Evidence Lower Bound (ELBO)

    68.5.1 Deriving the ELBO

    We have already seen the ELBO in Chapter 66 (§66.10.2). We re-derive it here more carefully, connecting it explicitly to the EM machinery the course has developed.

    For a single data point \(\mathbf{x}\), introduce any distribution \(q_\phi(\mathbf{z} \mid \mathbf{x})\) over the latent space. We can write the log-marginal as:

    \[\ln p_\theta(\mathbf{x}) = \ln \int p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z}) \, d\mathbf{z}\]

    Multiply and divide the integrand by \(q_\phi(\mathbf{z} \mid \mathbf{x})\) (assumed to have the same support as \(p(\mathbf{z})\)):

    \[= \ln \int q_\phi(\mathbf{z} \mid \mathbf{x}) \cdot \frac{p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})}{q_\phi(\mathbf{z} \mid \mathbf{x})} \, d\mathbf{z} = \ln \, \mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}\!\left[\frac{p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})}{q_\phi(\mathbf{z} \mid \mathbf{x})}\right]\]

    Apply Jensen's inequality (\(\ln\) is concave, so \(\ln \mathbb{E}[Y] \geq \mathbb{E}[\ln Y]\)):

    \[\geq \mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}\!\left[\ln \frac{p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})}{q_\phi(\mathbf{z} \mid \mathbf{x})}\right] \tag{68.9}\]

    Expanding the logarithm:

    \[= \mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] + \mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}\!\left[\ln \frac{p(\mathbf{z})}{q_\phi(\mathbf{z} \mid \mathbf{x})}\right]\]
    \[= \underbrace{\mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})]}_{\text{reconstruction term}} - \underbrace{D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p(\mathbf{z}))}_{\text{regularization term}} \tag{68.10}\]

    Definition 68.2 (Evidence Lower Bound). The ELBO for data point \(\mathbf{x}\) is:

    \[\mathcal{L}(\theta, \phi; \mathbf{x}) = \mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] - D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p(\mathbf{z})) \tag{68.11}\]

    We have established \(\ln p_\theta(\mathbf{x}) \geq \mathcal{L}(\theta, \phi; \mathbf{x})\) for any \(q_\phi\).

    68.5.2 The Gap Between the Log-Likelihood and the ELBO

    The ELBO is not a loose, arbitrary lower bound — it has a precise relationship to the true log-likelihood.

    Proposition 68.1. \(\ln p_\theta(\mathbf{x}) = \mathcal{L}(\theta, \phi; \mathbf{x}) + D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p_\theta(\mathbf{z} \mid \mathbf{x}))\).

    Proof. Starting from (68.10):

    \[\mathcal{L}(\theta, \phi; \mathbf{x}) = \mathbb{E}_{q_\phi}\!\left[\ln p_\theta(\mathbf{x} \mid \mathbf{z}) + \ln p(\mathbf{z}) - \ln q_\phi(\mathbf{z} \mid \mathbf{x})\right]\]

    Add and subtract \(\ln p_\theta(\mathbf{x})\) (a constant with respect to \(\mathbf{z}\)):

    \[= \ln p_\theta(\mathbf{x}) + \mathbb{E}_{q_\phi}\!\left[\ln p_\theta(\mathbf{x} \mid \mathbf{z}) + \ln p(\mathbf{z}) - \ln q_\phi(\mathbf{z} \mid \mathbf{x}) - \ln p_\theta(\mathbf{x})\right]\]
    \[= \ln p_\theta(\mathbf{x}) + \mathbb{E}_{q_\phi}\!\left[\ln \frac{p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})}{q_\phi(\mathbf{z} \mid \mathbf{x}) \, p_\theta(\mathbf{x})}\right]\]
    \[= \ln p_\theta(\mathbf{x}) + \mathbb{E}_{q_\phi}\!\left[\ln \frac{p_\theta(\mathbf{z} \mid \mathbf{x})}{q_\phi(\mathbf{z} \mid \mathbf{x})}\right]\]
    \[= \ln p_\theta(\mathbf{x}) - D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p_\theta(\mathbf{z} \mid \mathbf{x})) \qquad \blacksquare\]

    Therefore:

    \[\ln p_\theta(\mathbf{x}) = \mathcal{L}(\theta, \phi; \mathbf{x}) + D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p_\theta(\mathbf{z} \mid \mathbf{x})) \tag{68.12}\]

    Three consequences of (68.12):

    First, since \(D_{\text{KL}} \geq 0\), we confirm \(\ln p_\theta(\mathbf{x}) \geq \mathcal{L}(\theta, \phi; \mathbf{x})\).

    Second, the bound is tight when \(q_\phi(\mathbf{z} \mid \mathbf{x}) = p_\theta(\mathbf{z} \mid \mathbf{x})\), i.e., when the approximate posterior exactly equals the true posterior. In the GMM, this was achievable via Bayes' rule. In the VAE with a neural network decoder, it is generally not achievable (the true posterior may not belong to the Gaussian family), so the bound remains slightly loose.

    Third, maximizing the ELBO jointly over \(\theta\) and \(\phi\): - Maximizing over \(\theta\) increases \(\ln p_\theta(\mathbf{x})\) directly (makes the model fit the data better). - Maximizing over \(\phi\) reduces \(D_{\text{KL}}(q_\phi \| p_\theta)\) (makes the approximate posterior closer to the true posterior).

    Both effects are desirable and occur simultaneously when we gradient-ascend on \(\mathcal{L}(\theta, \phi; \mathbf{x})\).

    68.5.3 Interpretation of the Two ELBO Terms

    Term 1: Reconstruction. \(\mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})]\).

    This measures how well the decoder can reconstruct \(\mathbf{x}\) from latent codes \(\mathbf{z}\) drawn from the encoder's distribution. High reconstruction means: "for a typical encoding of \(\mathbf{x}\), the decoder assigns \(\mathbf{x}\) high probability." It is a measure of fidelity — the model is rewarded for producing outputs similar to the input.

    For continuous data with \(p_\theta(\mathbf{x} \mid \mathbf{z}) = \mathcal{N}(\mathbf{x} \mid f_\theta(\mathbf{z}), \sigma^2 \mathbf{I})\):

    \[\ln p_\theta(\mathbf{x} \mid \mathbf{z}) = -\frac{d}{2}\ln(2\pi\sigma^2) - \frac{1}{2\sigma^2}\|{\mathbf{x} - f_\theta(\mathbf{z})}\|^2\]

    So maximizing the reconstruction term is equivalent to minimizing the mean squared error \(\mathbb{E}_{q_\phi}[\|\mathbf{x} - f_\theta(\mathbf{z})\|^2]\) between the input and the decoded output. This is why VAEs are called autoencoders — they encode and then decode, and the reconstruction loss penalizes the round-trip error.

    Term 2: Regularization. \(-D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p(\mathbf{z}))\).

    This penalizes the encoder for deviating from the prior \(p(\mathbf{z}) = \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\). It acts as a regularizer on the latent space, pulling the approximate posteriors \(q_\phi(\mathbf{z} \mid \mathbf{x})\) toward the standard Gaussian. This has two effects:

    (a) Generation at test time. If \(q_\phi(\mathbf{z} \mid \mathbf{x}) \approx p(\mathbf{z})\) for all \(\mathbf{x}\), then sampling \(\mathbf{z} \sim p(\mathbf{z}) = \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\) at test time will produce \(\mathbf{z}\) values in regions that the decoder has seen during training, yielding meaningful outputs.

    (b) Smooth interpolation. The KL term prevents the encoder from memorizing (collapsing each \(\mathbf{x}\) to a delta function in \(\mathcal{Z}\)). If each \(\mathbf{x}\) were mapped to a distinct point in \(\mathcal{Z}\), sampling randomly from \(p(\mathbf{z})\) would land between training points, producing garbage. The regularization ensures neighboring regions of \(\mathcal{Z}\) decode to visually similar outputs.

    The tension. The two terms are in tension: the reconstruction term wants the encoder to map each \(\mathbf{x}\) to a sharp, precise \(\mathbf{z}\) (low variance), while the regularization term wants \(q_\phi(\mathbf{z} \mid \mathbf{x})\) to be broad and close to \(\mathcal{N}(\mathbf{0}, \mathbf{I}_k)\). The ELBO balances these competing pressures automatically.

    68.6 The VAE Architecture

    68.6.1 Representing Distributions with Neural Networks

    A neural network can represent a probability distribution in two ways, and the choice matters:

    Type A: Sample output. The network takes an input and directly outputs a sample from the target distribution. Example: the GAN generator \(G_\theta(\mathbf{z})\) outputs a sample \(\tilde{\mathbf{x}} \sim p_\theta(\mathbf{x} \mid \mathbf{z})\).

    Type B: Parameter output. The network takes an input and outputs the parameters of a parametric distribution. The user then samples from that distribution using a known primitive. Example: a classifier with softmax output gives \((\pi_1, \ldots, \pi_K)\), the parameters of a categorical distribution over class labels.

    The VAE encoder uses Type B. We parameterize \(q_\phi(\mathbf{z} \mid \mathbf{x}) = \mathcal{N}(\mathbf{z} \mid \boldsymbol{\mu}_\phi(\mathbf{x}), \text{diag}(\boldsymbol{\sigma}^2_\phi(\mathbf{x})))\) — a diagonal-covariance Gaussian whose parameters depend on \(\mathbf{x}\). The encoder network \(\text{Enc}_\phi : \mathbb{R}^d \to \mathbb{R}^k \times \mathbb{R}^k_{>0}\) outputs:

    \[\text{Enc}_\phi(\mathbf{x}) = (\boldsymbol{\mu}_\phi(\mathbf{x}), \boldsymbol{\sigma}^2_\phi(\mathbf{x})) \tag{68.13}\]

    where \(\boldsymbol{\mu}_\phi(\mathbf{x}) \in \mathbb{R}^k\) (mean vector, unconstrained, with linear output activations) and \(\boldsymbol{\sigma}^2_\phi(\mathbf{x}) \in \mathbb{R}^k_{>0}\) (variance vector, constrained positive, e.g., via softplus or \(e^{\cdot}\) output activations). In practice, it is common to output \(\log \boldsymbol{\sigma}^2_\phi(\mathbf{x})\) and exponentiate, which avoids numerical issues.

    The encoder therefore outputs a full Gaussian distribution, not a single point. For each data point \(\mathbf{x}\), we get a distinct Gaussian distribution \(q_\phi(\mathbf{z} \mid \mathbf{x})\) in the latent space.

    The VAE decoder. The decoder \(\text{Dec}_\theta : \mathbb{R}^k \to \mathcal{X}\) takes a latent vector \(\mathbf{z}\) and outputs the parameters of \(p_\theta(\mathbf{x} \mid \mathbf{z})\). For the Gaussian decoder:

    \[\text{Dec}_\theta(\mathbf{z}) = f_\theta(\mathbf{z}), \quad p_\theta(\mathbf{x} \mid \mathbf{z}) = \mathcal{N}(\mathbf{x} \mid f_\theta(\mathbf{z}), \sigma^2 \mathbf{I}) \tag{68.14}\]

    68.6.2 The Data Flow

    For a single training point \(\mathbf{x}\), the forward pass of a VAE is:

    \[\mathbf{x} \xrightarrow{\text{Enc}_\phi} (\boldsymbol{\mu}_\phi, \boldsymbol{\sigma}^2_\phi) \xrightarrow{\text{sample}} \mathbf{z} \sim \mathcal{N}(\boldsymbol{\mu}_\phi, \text{diag}(\boldsymbol{\sigma}^2_\phi)) \xrightarrow{\text{Dec}_\theta} f_\theta(\mathbf{z}) \approx \mathbf{x}\]

    Compute the ELBO:

    \[\mathcal{L}(\theta, \phi; \mathbf{x}) = \mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] - D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p(\mathbf{z}))\]

    Back-propagate the gradient of \(-\mathcal{L}\) (we minimize the negative ELBO) through both \(\theta\) and \(\phi\) jointly.

    68.6.3 The Sampling Problem: Non-Differentiability

    There is a fundamental obstruction to naively computing \(\nabla_\phi \mathcal{L}\). The first ELBO term involves:

    \[\mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] = \int q_\phi(\mathbf{z} \mid \mathbf{x}) \ln p_\theta(\mathbf{x} \mid \mathbf{z}) \, d\mathbf{z}\]

    We estimate this by Monte Carlo: draw \(\mathbf{z} \sim q_\phi(\mathbf{z} \mid \mathbf{x})\), then evaluate \(\ln p_\theta(\mathbf{x} \mid \mathbf{z})\). But the sampling step \(\mathbf{z} \sim q_\phi(\mathbf{z} \mid \mathbf{x})\) depends on \(\phi\). Sampling is a stochastic operation, and stochastic operations are not differentiable in the standard sense — there is no chain rule through a random number generator.

    Formally: define \(F(\phi) = \mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[f(\mathbf{z})]\) for some function \(f\). We need \(\nabla_\phi F(\phi)\). We cannot write \(\nabla_\phi F(\phi) = \mathbb{E}_{q_\phi}[\nabla_\phi f(\mathbf{z})]\) because \(\nabla_\phi f(\mathbf{z}) = 0\) (f does not depend on \(\phi\)). The \(\phi\)-dependence is entirely in the distribution we are sampling from, not in the integrand.

    One approach is the REINFORCE / log-derivative trick (also called score function estimator):

    \[\nabla_\phi \mathbb{E}_{q_\phi}[f(\mathbf{z})] = \mathbb{E}_{q_\phi}[f(\mathbf{z}) \, \nabla_\phi \ln q_\phi(\mathbf{z} \mid \mathbf{x})]\]

    This is unbiased but has extremely high variance for complex models, making it impractical.

    The VAE uses a far superior approach: the reparameterization trick.

    68.7 The Reparameterization Trick

    68.7.1 The Core Idea

    The reparameterization trick exploits the following observation: sampling from \(\mathcal{N}(\boldsymbol{\mu}_\phi, \text{diag}(\boldsymbol{\sigma}^2_\phi))\) is equivalent to a deterministic transformation of a sample from \(\mathcal{N}(\mathbf{0}, \mathbf{I}_k)\).

    Proposition 68.2 (Reparameterization). Let \(\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\). Define:

    \[\mathbf{z} = \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon} \tag{68.15}\]

    where \(\odot\) denotes element-wise multiplication and \(\boldsymbol{\sigma}_\phi = \sqrt{\boldsymbol{\sigma}^2_\phi}\) (element-wise). Then \(\mathbf{z} \sim \mathcal{N}(\boldsymbol{\mu}_\phi(\mathbf{x}), \text{diag}(\boldsymbol{\sigma}^2_\phi(\mathbf{x})))\).

    Proof. Each component: \(z_j = \mu_{\phi,j}(\mathbf{x}) + \sigma_{\phi,j}(\mathbf{x}) \cdot \epsilon_j\) with \(\epsilon_j \sim \mathcal{N}(0,1)\). By the affine transformation property of Gaussians, \(z_j \sim \mathcal{N}(\mu_{\phi,j}, \sigma_{\phi,j}^2)\). Independence across \(j\) gives the diagonal-covariance Gaussian. \(\blacksquare\)

    68.7.2 Why This Solves the Gradient Problem

    With (68.15), the Monte Carlo estimate of the first ELBO term becomes:

    \[\mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] = \mathbb{E}_{\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})}[\ln p_\theta(\mathbf{x} \mid \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon})] \tag{68.16}\]

    The expectation is now over \(\boldsymbol{\epsilon}\), which does not depend on \(\phi\). The \(\phi\)-dependence is entirely inside the deterministic function \(\boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon}\), which is differentiable with respect to \(\phi\) via standard backpropagation.

    We can therefore pass the gradient through the sampling step:

    \[\nabla_\phi \, \mathbb{E}_{q_\phi}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] = \mathbb{E}_{\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})}\!\left[\nabla_\phi \ln p_\theta(\mathbf{x} \mid \boldsymbol{\mu}_\phi + \boldsymbol{\sigma}_\phi \odot \boldsymbol{\epsilon})\right] \tag{68.17}\]

    Estimating (68.17) with a single sample \(\boldsymbol{\epsilon}^{(1)} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})\):

    \[\nabla_\phi \, \mathbb{E}_{q_\phi}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] \approx \nabla_\phi \ln p_\theta(\mathbf{x} \mid \boldsymbol{\mu}_\phi + \boldsymbol{\sigma}_\phi \odot \boldsymbol{\epsilon}^{(1)})\]

    which is a low-variance, unbiased gradient estimator computable by backpropagation through \(\boldsymbol{\mu}_\phi\) and \(\boldsymbol{\sigma}_\phi\).

    The reparameterization trick is the Push-Forward Principle of Chapter 66 in action: we have rewritten the sampling process \(\mathbf{z} \sim q_\phi(\mathbf{z} \mid \mathbf{x})\) as a deterministic transformation \(\mathbf{z} = g_\phi(\boldsymbol{\epsilon}; \mathbf{x})\) of a noise variable \(\boldsymbol{\epsilon}\), which is independent of \(\phi\).

    68.7.3 The Closed-Form KL Term

    The second ELBO term \(D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p(\mathbf{z}))\) requires no sampling — it has an exact closed form when both distributions are Gaussian.

    Proposition 68.3 (KL Between Gaussians). Let \(q = \mathcal{N}(\boldsymbol{\mu}, \text{diag}(\boldsymbol{\sigma}^2))\) and \(p = \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\). Then:

    \[D_{\text{KL}}(q \,\|\, p) = \frac{1}{2} \sum_{j=1}^k \left(\mu_j^2 + \sigma_j^2 - 1 - \ln \sigma_j^2\right) \tag{68.18}\]

    Proof. By independence of dimensions, \(D_{\text{KL}}(q \| p) = \sum_{j=1}^k D_{\text{KL}}(\mathcal{N}(\mu_j, \sigma_j^2) \,\|\, \mathcal{N}(0, 1))\).

    For a single dimension, compute:

    \[D_{\text{KL}}(\mathcal{N}(\mu, \sigma^2) \,\|\, \mathcal{N}(0, 1)) = \int \mathcal{N}(z; \mu, \sigma^2) \ln \frac{\mathcal{N}(z; \mu, \sigma^2)}{\mathcal{N}(z; 0, 1)} \, dz\]
    \[= \int \mathcal{N}(z; \mu, \sigma^2) \left[-\frac{(z-\mu)^2}{2\sigma^2} - \frac{1}{2}\ln(2\pi\sigma^2) + \frac{z^2}{2} + \frac{1}{2}\ln(2\pi)\right] dz\]

    Evaluate each integral using \(\mathbb{E}_q[(z-\mu)^2] = \sigma^2\), \(\mathbb{E}_q[z^2] = \sigma^2 + \mu^2\):

    \[= -\frac{\sigma^2}{2\sigma^2} - \frac{1}{2}\ln(2\pi\sigma^2) + \frac{\sigma^2 + \mu^2}{2} + \frac{1}{2}\ln(2\pi)\]
    \[= -\frac{1}{2} - \frac{1}{2}\ln\sigma^2 + \frac{\sigma^2 + \mu^2}{2}\]
    \[= \frac{1}{2}\left(\mu^2 + \sigma^2 - 1 - \ln\sigma^2\right) \qquad \blacksquare\]

    This expression is differentiable with respect to \(\mu_j\) and \(\sigma_j^2\), so the gradient \(\nabla_\phi D_{\text{KL}}\) is available in closed form (just differentiate (68.18) through \(\boldsymbol{\mu}_\phi(\mathbf{x})\) and \(\boldsymbol{\sigma}^2_\phi(\mathbf{x})\)).

    68.7.4 The Full Differentiable ELBO

    Combining the reparameterized first term and the closed-form second term, the ELBO for a single data point \(\mathbf{x}\) with a single noise sample \(\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\) becomes:

    \[\mathcal{L}(\theta, \phi; \mathbf{x}) \approx \ln p_\theta(\mathbf{x} \mid \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon}) - \frac{1}{2}\sum_{j=1}^k \left(\mu_{\phi,j}^2 + \sigma_{\phi,j}^2 - 1 - \ln\sigma_{\phi,j}^2\right) \tag{68.19}\]

    Both terms are differentiable with respect to \(\theta\) and \(\phi\). The training objective is:

    \[\max_{\theta, \phi} \frac{1}{n}\sum_{i=1}^n \mathcal{L}(\theta, \phi; \mathbf{x}^{(i)}) \tag{68.20}\]

    implemented by standard mini-batch stochastic gradient ascent.

    68.8 The VAE Training Algorithm

    Algorithm 68.1: VAE Training

    Input: Dataset \(\mathcal{D}\), latent dimension \(k\), batch size \(m\), learning rate \(\eta\).

    Initialize: Encoder parameters \(\phi\), decoder parameters \(\theta\), randomly.

    For each training epoch:

    For each mini-batch \(\{\mathbf{x}^{(i)}\}_{i=1}^m \subset \mathcal{D}\):

    1. Encode: Compute \((\boldsymbol{\mu}_\phi^{(i)}, \log \boldsymbol{\sigma}_\phi^{(2,i)}) = \text{Enc}_\phi(\mathbf{x}^{(i)})\) for each \(i\).
    2. Reparameterize: Sample \(\boldsymbol{\epsilon}^{(i)} \sim \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\) and compute \(\mathbf{z}^{(i)} = \boldsymbol{\mu}_\phi^{(i)} + \boldsymbol{\sigma}_\phi^{(i)} \odot \boldsymbol{\epsilon}^{(i)}\).
    3. Decode: Compute \(f_\theta(\mathbf{z}^{(i)}) = \text{Dec}_\theta(\mathbf{z}^{(i)})\) for each \(i\).
    4. Compute ELBO: $\(\hat{\mathcal{L}} = \frac{1}{m}\sum_{i=1}^m \left[\ln p_\theta(\mathbf{x}^{(i)} \mid \mathbf{z}^{(i)}) - \frac{1}{2}\sum_{j=1}^k\!\left((\mu_{\phi,j}^{(i)})^2 + (\sigma_{\phi,j}^{(i)})^2 - 1 - \ln(\sigma_{\phi,j}^{(i)})^2\right)\right]\)$
    5. Update: \((\theta, \phi) \leftarrow (\theta, \phi) + \eta \nabla_{\theta,\phi} \hat{\mathcal{L}}\) (joint gradient ascent).

    At inference (generation): Draw \(\mathbf{z} \sim \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\), compute \(\tilde{\mathbf{x}} = f_{\theta^*}(\mathbf{z})\) (or sample from \(p_{\theta^*}(\mathbf{x} \mid \mathbf{z})\)).

    At inference (embedding): Pass \(\mathbf{x}\) through encoder, return \(\boldsymbol{\mu}_{\phi^*}(\mathbf{x})\) as the embedding (or sample \(\mathbf{z} \sim q_{\phi^*}(\mathbf{z} \mid \mathbf{x})\)).

    Note on the gradient flow. During the forward pass, the computation graph is:

    \[\phi \to (\boldsymbol{\mu}_\phi, \boldsymbol{\sigma}_\phi) \to \mathbf{z} = \boldsymbol{\mu}_\phi + \boldsymbol{\sigma}_\phi \odot \boldsymbol{\epsilon} \to f_\theta(\mathbf{z}) \to \ln p_\theta(\mathbf{x} \mid \mathbf{z})\]

    The edge \((\boldsymbol{\mu}_\phi, \boldsymbol{\sigma}_\phi) \to \mathbf{z}\) is a deterministic arithmetic operation (not a sampling operation), because \(\boldsymbol{\epsilon}\) is treated as a constant input during backpropagation. The gradient therefore flows freely from the reconstruction loss all the way back to \(\phi\) through the encoder.

    68.9 Worked Example: VAE on Univariate Data

    Setup. Let \(d = k = 1\) (both data and latent are scalar). Prior: \(p(z) = \mathcal{N}(0, 1)\). Target: \(p(x) = \mathcal{N}(5, 4)\) (mean 5, variance 4). Decoder: \(p_\theta(x \mid z) = \mathcal{N}(x \mid \theta_1 z + \theta_2, 1)\) (affine, unit variance). Encoder: \(q_\phi(z \mid x) = \mathcal{N}(z \mid \phi_1 x + \phi_0, e^{\phi_2})\) (affine mean, learned log-variance).

    ELBO calculation. The first term:

    \[\mathbb{E}_{q_\phi(z \mid x)}[\ln p_\theta(x \mid z)] = -\frac{1}{2}\ln(2\pi) - \frac{1}{2}\mathbb{E}_{q_\phi}\!\left[(x - \theta_1 z - \theta_2)^2\right]\]
    \[= -\frac{1}{2}\ln(2\pi) - \frac{1}{2}\left[(x - \theta_1 \mu_\phi - \theta_2)^2 + \theta_1^2 \sigma_\phi^2\right]\]

    where \(\mu_\phi = \phi_1 x + \phi_0\) and \(\sigma_\phi^2 = e^{\phi_2}\). (We used \(\mathbb{E}_q[z^2] = \mu_\phi^2 + \sigma_\phi^2\).)

    The second term:

    \[D_{\text{KL}}(q_\phi(z \mid x) \,\|\, p(z)) = \frac{1}{2}\left(\mu_\phi^2 + \sigma_\phi^2 - 1 - \ln\sigma_\phi^2\right) = \frac{1}{2}\left(\mu_\phi^2 + e^{\phi_2} - 1 - \phi_2\right)\]

    At optimum. The true posterior is \(p_\theta(z \mid x) \propto p_\theta(x \mid z) p(z) = \mathcal{N}(x \mid \theta_1 z + \theta_2, 1) \cdot \mathcal{N}(z \mid 0, 1)\). Completing the square in \(z\):

    \[\ln p_\theta(z \mid x) \propto -\frac{(x - \theta_1 z - \theta_2)^2}{2} - \frac{z^2}{2} = -\frac{(1 + \theta_1^2)}{2}\left(z - \frac{\theta_1(x-\theta_2)}{1+\theta_1^2}\right)^2 + \text{const}\]

    So \(p_\theta(z \mid x) = \mathcal{N}\!\left(z \mid \frac{\theta_1(x-\theta_2)}{1+\theta_1^2}, \frac{1}{1+\theta_1^2}\right)\).

    At the globally optimal VAE: - \(\theta_1 = 2\), \(\theta_2 = 5\) (so \(p_\theta(x \mid z) = \mathcal{N}(x \mid 2z + 5, 1)\), and \(p_\theta(x) = \mathcal{N}(5, 4+1) = \mathcal{N}(5, 5)\) — approximately right). - The encoder learns \(q_\phi(z \mid x) = \mathcal{N}(z \mid \frac{2(x-5)}{5}, \frac{1}{5})\), matching the true posterior.

    This toy example illustrates that the encoder and decoder co-adapt: the decoder learns how to expand the latent space to match the data distribution, and the encoder learns the inverse map.

    68.10 Inference with a Trained VAE

    A trained VAE supports two distinct inference modes.

    68.10.1 Generation (Decoder Mode)

    To produce a new sample \(\tilde{\mathbf{x}}\) not in \(\mathcal{D}\):

    1. Draw \(\mathbf{z} \sim p(\mathbf{z}) = \mathcal{N}(\mathbf{0}, \mathbf{I}_k)\).
    2. Pass through the decoder: \(\tilde{\mathbf{x}} = f_{\theta^*}(\mathbf{z})\) (for continuous data with MSE reconstruction).

    The encoder \(\text{Enc}_\phi\) is not used during generation. Generation is purely decoder-driven, like a GAN — the difference is that the VAE's decoder was trained via ELBO maximization rather than adversarial training.

    68.10.2 Embedding Extraction (Encoder Mode)

    To obtain a latent representation for a data point \(\mathbf{x}\):

    1. Pass \(\mathbf{x}\) through the encoder: \((\boldsymbol{\mu}_{\phi^*}(\mathbf{x}), \boldsymbol{\sigma}^2_{\phi^*}(\mathbf{x})) = \text{Enc}_{\phi^*}(\mathbf{x})\).
    2. Use \(\boldsymbol{\mu}_{\phi^*}(\mathbf{x}) \in \mathbb{R}^k\) as the embedding (a deterministic \(k\)-dimensional vector).
    3. Alternatively, sample \(\mathbf{z} \sim q_{\phi^*}(\mathbf{z} \mid \mathbf{x})\) and use \(\mathbf{z}\) as a stochastic embedding.

    The mean \(\boldsymbol{\mu}_{\phi^*}(\mathbf{x})\) is the most natural embedding: it is the "center" of the posterior distribution of \(\mathbf{z}\) given \(\mathbf{x}\). Because the ELBO has regularized the latent space toward \(\mathcal{N}(\mathbf{0}, \mathbf{I}_k)\), these embeddings are well-organized: data points that are semantically similar (e.g., different images of the digit "3") will have nearby \(\boldsymbol{\mu}\) values in \(\mathbb{R}^k\).

    68.10.3 Latent Space Interpolation

    A consequence of the regularized latent space is that linear interpolation in \(\mathcal{Z}\) corresponds to smooth interpolation in \(\mathcal{X}\). Given two data points \(\mathbf{x}_A\) and \(\mathbf{x}_B\) with latent means \(\boldsymbol{\mu}_A = \boldsymbol{\mu}_{\phi^*}(\mathbf{x}_A)\) and \(\boldsymbol{\mu}_B = \boldsymbol{\mu}_{\phi^*}(\mathbf{x}_B)\):

    \[\tilde{\mathbf{x}}_t = f_{\theta^*}((1-t)\boldsymbol{\mu}_A + t\boldsymbol{\mu}_B), \quad t \in [0,1]\]

    produces a smooth morphing from \(\mathbf{x}_A\) to \(\mathbf{x}_B\) as \(t\) goes from 0 to 1. This would not work in a GAN because there is no encoder to map \(\mathbf{x}_A\) and \(\mathbf{x}_B\) to latent codes.

    68.11 Latent Variable Models as the Foundation for Modern Generative AI

    The lecture mentioned a critical observation that warrants expansion: the encoder-decoder structure of latent variable models is not just a self-contained model class. It is the infrastructure on which the most capable modern generative systems are built.

    68.11.1 The Two-Stage Architecture

    Modern image generation systems (as of the time of this course) operate in two stages:

    Stage 1: Build a latent space. Train a latent variable model (typically a VQ-VAE or VAE variant) on the raw data. This gives a trained encoder \(\text{Enc}_\phi : \mathcal{X} \to \mathcal{Z}\) and decoder \(\text{Dec}_\theta : \mathcal{Z} \to \mathcal{X}\). The latent space \(\mathcal{Z}\) is much lower-dimensional than \(\mathcal{X}\): for images, \(d\) might be \(786432\) (for a \(512 \times 512 \times 3\) image) while \(k\) might be \(4096\) — a 192-fold compression.

    Stage 2: Build a generative model on \(\mathcal{Z}\). Train a powerful generative model (a diffusion model, an autoregressive model, etc.) not on raw data \(\mathbf{x} \in \mathcal{X}\), but on latent codes \(\mathbf{z} = \text{Enc}_\phi(\mathbf{x}) \in \mathcal{Z}\). Because \(\mathcal{Z}\) is far smaller and more structured than \(\mathcal{X}\), this is much more computationally tractable.

    Inference: To generate a new image: 1. Sample \(\hat{\mathbf{z}}\) from the Stage-2 generative model. 2. Decode: \(\tilde{\mathbf{x}} = \text{Dec}_\theta(\hat{\mathbf{z}})\).

    The raw image \(\tilde{\mathbf{x}}\) is displayed to the user, who never sees the latent code \(\hat{\mathbf{z}}\).

    This architecture is called Latent Diffusion (Rombach et al., 2022), and it underlies most modern text-to-image systems. The text prompt conditions the Stage-2 generative model (making it a conditional generative model on \(\mathcal{Z}\)), and the decoder then renders the output in pixel space.

    The conceptual contribution of this chapter is that the encoder-decoder pair (VAE/VQ-VAE) provides the vocabulary in which high-level generative models speak. The Stage-2 model generates meaning (in latent space); the decoder translates meaning into perception (in data space).

    68.12 VAE vs. GAN: A Mathematical Comparison

    Having studied both families in depth, we can now compare them precisely along the dimensions that matter.

    Divergence minimized. The GAN minimizes \(D_{\text{JS}}(p \| p_\theta)\) (or the Wasserstein distance, for WGANs). The VAE maximizes the ELBO, which is equivalent to minimizing:

    \[\text{VAE objective} \Leftrightarrow \min_{\theta, \phi} \left[-\mathbb{E}_{q_\phi(\mathbf{z} \mid \mathbf{x})}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] + D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \,\|\, p(\mathbf{z}))\right]\]

    This does not directly minimize \(D_{\text{KL}}(p \| p_\theta)\) — it minimizes a bound on it, with the bound gap equal to \(D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \| p_\theta(\mathbf{z} \mid \mathbf{x}))\) (the quality of the approximate posterior).

    Density tractability. The GAN provides no \(\ln p_\theta(\mathbf{x})\). The VAE provides \(\mathcal{L}(\theta, \phi; \mathbf{x})\), a lower bound on \(\ln p_\theta(\mathbf{x})\), which is useful for evaluation and anomaly detection.

    Sample quality vs. sample diversity. GANs are typically praised for producing sharper, more photorealistic samples (they directly minimize a distributional divergence without a reconstruction term). VAEs tend to produce slightly blurrier samples because the Gaussian decoder encourages averaging (the MSE reconstruction minimizes the expected squared error, which corresponds to the posterior mean). However, VAEs produce more diverse samples and are less prone to mode collapse.

    Training stability. VAE training is a standard gradient ascent problem on a well-defined scalar objective — stable and converging. GAN training is a minimax problem without convergence guarantees — prone to oscillation and collapse.

    Embedding extraction. GANs: not available (decoder-only). VAEs: directly available via the encoder.

    GAN VAE
    Objective Minimax (saddle point) ELBO maximization (standard opt.)
    Divergence \(D_{\text{JS}}\) (or Wasserstein) Bound on \(D_{\text{KL}}\)

    68.13 Summary

    This chapter developed the theory of latent variable generative models, culminating in the Variational Autoencoder.

    A latent variable model posits a hidden variable \(\mathbf{z}\) with prior \(p(\mathbf{z})\) and a conditional decoder \(p_\theta(\mathbf{x} \mid \mathbf{z})\). The marginal \(p_\theta(\mathbf{x}) = \int p_\theta(\mathbf{x} \mid \mathbf{z}) p(\mathbf{z}) d\mathbf{z}\) is the generative distribution. The model is an encoder-decoder pair: the encoder is the posterior \(p_\theta(\mathbf{z} \mid \mathbf{x})\) (feature extractor), and the decoder is \(p_\theta(\mathbf{x} \mid \mathbf{z})\) (generator).

    The Gaussian Mixture Model is the simplest instance. Its discrete latent variable makes sampling a two-step process: draw a component index via the inverse-CDF method, then draw from the corresponding Gaussian. Its limitation is that the tractable posterior requires a closed-form conditional, restricting the model to simple likelihoods.

    The VAE generalizes the latent variable model to neural network decoders by: - Introducing an approximate posterior (encoder) \(q_\phi(\mathbf{z} \mid \mathbf{x})\) to replace the intractable true posterior. - Deriving the ELBO via Jensen's inequality: \(\ln p_\theta(\mathbf{x}) \geq \mathcal{L}(\theta, \phi; \mathbf{x}) = \mathbb{E}_{q_\phi}[\ln p_\theta(\mathbf{x} \mid \mathbf{z})] - D_{\text{KL}}(q_\phi \| p)\). - The gap between the ELBO and \(\ln p_\theta(\mathbf{x})\) is \(D_{\text{KL}}(q_\phi(\mathbf{z} \mid \mathbf{x}) \| p_\theta(\mathbf{z} \mid \mathbf{x}))\); maximizing the ELBO simultaneously tightens this gap. - Using the reparameterization trick to make the ELBO differentiable: \(\mathbf{z} = \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon}\), \(\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})\), converts the stochastic sampling step into a deterministic transformation, enabling standard backpropagation. - The KL term has a closed form for Gaussian \(q_\phi\) and Gaussian \(p\): \(D_{\text{KL}} = \frac{1}{2}\sum_j(\mu_j^2 + \sigma_j^2 - 1 - \ln\sigma_j^2)\).

    At inference, the VAE supports both generation (sample \(\mathbf{z} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})\), decode) and embedding extraction (encode \(\mathbf{x}\), return \(\boldsymbol{\mu}_\phi(\mathbf{x})\)).

    The encoder-decoder structure of latent variable models is the foundation of modern image generation pipelines: a VAE compresses data into a structured latent space, and a downstream generative model (diffusion, autoregressive) operates in that latent space, with the decoder rendering final outputs.
