    Mathematical Foundations of Machine Learning

    Chapter 15: Discrete Support Manifolds, One-Hot Representations, and Constrained Categorical MLE

    15.1 The Strategic Transition: Continuous vs. Discrete Support

    In the previous chapter, we derived the Maximum Likelihood Estimation (MLE) protocol for a continuous Multivariate Gaussian distribution. We proved that calculating a standard vector mean average is the mathematically exact method for minimizing the Kullback-Leibler (KL) divergence to nature when your data noise follows a smooth bell curve.

    Now, we shift our focus to Discrete Support Manifolds.

    When dealing with categorical attributes—such as rolling a multi-sided die, identifying a distinct word token from a vocabulary list, or sorting an image into discrete diagnostic bins—the data coordinates do not sit on a continuous line. They exist as isolated, non-numeric categorical states.

    Deconstructing a Common Misconception: Multinoulli vs. Multinomial

    The professor halts the lecture to clear up a major piece of terminology that often trips up practitioners:

    The Categorical Rule: > Do not call a single multi-outcome trial a "Binomial" or "Multinomial" distribution. * A Binomial or Multinomial distribution tracks the accumulated counts of outcomes gathered across multiple independent trials. * A single trial with exactly two outcomes is a Bernoulli distribution. * A single trial with \(M\) distinct categorical outcomes (\(M > 2\)) is a Multinoulli (or Categorical) distribution.

    Because our dataset \(D\) is composed of \(N\) independent, single-observation trials, we are running parametric density estimation over a Categorical Space.

    15.2 The Problem Setup: The General Discrete Case

    We are given an empirical dataset \(D\) containing \(N\) independent and identically distributed (i.i.d.) scalar observations:

    \[D = \{v_1, v_2, \dots, v_N\} \sim \text{i.i.d. } p_{\text{true}}(v)\]

    The underlying random variable \(V\) can realize into exactly one of \(M\) distinct, non-numeric categorical states from a fixed support alphabet \(\mathcal{A}\):

    \[\mathcal{A} = \{a_1, a_2, \dots, a_M\}\]

    Each distinct state \(a_j\) is governed by a true, hidden background probability parameter \(p_j\):

    \[P(V = a_j) = p_j\]

    Our optimization parameter vector \(\vec{\theta}\) compiles these individual class probabilities into a single column array:

    \[\vec{\theta} = [p_1, p_2, \dots, p_M]^T\]

    Because the total probability mass of any valid system must conserve volume, these parameters are bound by a strict sum-to-one structural constraint:

    \[\sum_{j=1}^{M} p_j = 1\]

    Due to this structural constraint, the parameter vector does not have \(M\) complete degrees of freedom; it possesses exactly \(M - 1\) independent variables. If you know the values of the first \(M-1\) parameters, the final parameter is deterministically locked: \(p_M = 1 - \sum_{j=1}^{M-1} p_j\).

    15.3 The One-Hot Representation Mapping (The Algebraic Hook)

    To run maximum likelihood estimation, we must compute partial derivatives across our model parameters. However, you cannot run calculus transitions or compute algebraic derivatives over raw, non-numeric categories like \(\mathcal{A} = \{\text{"Heads"}, \text{"Tails"}\}\) or \(\mathcal{A} = \{\text{"Cough"}, \text{"Fever"}, \text{"Rash"}\}\).

    To create a clean algebraic hook for calculus, we map each scalar discrete observation \(v_i\) into an auxiliary, \(M\)-dimensional binary coordinate vector \(\vec{z}_i\):

    \[\vec{z}_i = [z_{i,1}, z_{i,2}, \dots, z_{i,M}]^T \in \{0, 1\}^M\]

    This transformation is governed by a strict structural conditional switch:

    \[z_{i,j} = \mathbb{I}(v_i = a_j) = \begin{cases} 1 & \text{if } v_i = a_j \\ 0 & \text{if } v_i \neq a_j \end{cases}\]

    The Operational Example: A 6-Sided Die Roll

    Let our support alphabet represent a standard 6-sided die: \(\mathcal{A} = \{1, 2, 3, 4, 5, 6\}\), meaning \(M = 6\). Suppose we roll the die \(N = 4\) times and harvest a tiny raw scalar dataset:

    \[D_{\text{raw}} = \{3, \,\, 1, \,\, 4, \,\, 3\}\]

    We pull each scalar coordinate through our indicator switch to construct our new one-hot encoded dataset ledger:

    \[\vec{z}_1 = \text{OneHot}(3) = [0, \,\, 0, \,\, 1, \,\, 0, \,\, 0, \,\, 0]^T\]
    \[\vec{z}_2 = \text{OneHot}(1) = [1, \,\, 0, \,\, 0, \,\, 0, \,\, 0, \,\, 0]^T\]
    \[\vec{z}_3 = \text{OneHot}(4) = [0, \,\, 0, \,\, 0, \,\, 1, \,\, 0, \,\, 0]^T\]
    \[\vec{z}_4 = \text{OneHot}(3) = [0, \,\, 0, \,\, 1, \,\, 0, \,\, 0, \,\, 0]^T\]

    Every individual data vector \(\vec{z}_i\) acts as a spatial pointer containing exactly one "hot" index (\(1\)) representing the active realization, while the remaining \(M-1\) positions are "cold" (\(0\)). This guarantees that the sum of the elements within any single data row is always exactly \(1\):

    \[\sum_{j=1}^{M} z_{i,j} = 1 \quad \forall \,\, i\]

    15.4 The Unified Differentiable PMF Formula

    Now that our dataset has been transformed into binary vectors, we can write down a single, unified Probability Mass Function (PMF) that encapsulates the entire categorical support space in a single line.

    Using our one-hot coordinate components as exponents, the density profile for a single observation is written as:

    \[p_{\vec{\theta}}(\vec{z}_i) = \prod_{j=1}^{M} p_j^{z_{i,j}} = p_1^{z_{i,1}} \cdot p_2^{z_{i,2}} \cdots p_M^{z_{i,M}}\]

    Why This Product Trick Works

    This formula functions as an elegant mathematical choice selector.

    Suppose an individual patient's observation vector corresponds to the second category (\(v_i = a_2\)). Its one-hot coordinates evaluate to \(z_{i,2} = 1\), and \(z_{i,j} = 0\) for all \(j \neq 2\). Let us plug these components directly into our product formula:

    \[p_{\vec{\theta}}(\vec{z}_i) = p_1^{0} \cdot p_2^{1} \cdot p_3^{0} \cdots p_M^{0}\]

    Because any non-zero real number raised to the power of \(0\) equals \(1\), every inactive parameter in the product collapses to unity, leaving the active parameter completely untouched:

    \[p_{\vec{\theta}}(\vec{z}_i) = 1 \cdot p_2 \cdot 1 \cdots 1 = p_2\]

    The product of powers formulation allows us to write a continuous, fully differentiable algebraic function that perfectly tracks discrete categorical selections.

    15.5 Formulating the Empirical Log-Likelihood

    Using our universal i.i.d. conversion engine, the global log-likelihood across our entire training ledger of \(N\) one-hot encoded rows is the summation of individual log-densities:

    \[\mathcal{L}(\vec{\theta}) = \frac{1}{N} \sum_{i=1}^{N} \ln p_{\vec{\theta}}(\vec{z}_i)\]

    Substituting our unified product PMF formula into this ledger yields:

    \[\mathcal{L}(\vec{\theta}) = \frac{1}{N} \sum_{i=1}^{N} \ln \left[ \prod_{j=1}^{M} p_j^{z_{i,j}} \right]\]

    Using the foundational operational properties of logarithms (\(\ln(\prod a_j) = \sum \ln a_j\)), we bring the vector exponent \(z_{i,j}\) down to the front, transforming the nested product into a clean double-summation matrix calculation:

    \[\mathcal{L}(\vec{\theta}) = \frac{1}{N} \sum_{i=1}^{N} \sum_{j=1}^{M} z_{i,j} \ln p_j\]

    15.6 Constrained Optimization via the Lagrangian Manifold

    The professor poses an essential question: Can we take the partial derivative of this log-likelihood function with respect to \(p_j\), set it to zero, and extract our parameters directly?

    The Constraint Rule: No. If you run an unconstrained optimization algorithm over this log-likelihood equation, the calculus will break. Because the log function grows monotonically, an unconstrained algorithm will attempt to maximize the score by driving every probability parameter \(p_j\) toward positive infinity (\(\infty\)).

    To force the parameters to remain valid probabilities, we must convert our problem into a Constrained Optimization Problem. We bound our landscape using our structural sum-to-one constraint:

    \[\text{Maximize } \mathcal{L}(\vec{\theta}) = \frac{1}{N} \sum_{i=1}^{N} \sum_{j=1}^{M} z_{i,j} \ln p_j \quad \text{Subject to } \sum_{j=1}^{M} p_j = 1\]

    To solve this smoothly, we invoke the method of Lagrange Multipliers. We construct an unconstrained surrogate function—the Lagrangian (\(\mathcal{L}_{\text{Lag}}\))—by subtracting our constraint boundary scaled by a new optimization variable, the Lagrange multiplier \(\lambda\):

    \[\mathcal{L}_{\text{Lag}}(\vec{\theta}, \lambda) = \frac{1}{N} \sum_{i=1}^{N} \sum_{j=1}^{M} z_{i,j} \ln p_j - \lambda \left( \sum_{j=1}^{M} p_j - 1 \right)\]

    (Note: The full derivation includes additional inequality constraints to keep \(0 \leq p_j \leq 1\). However, because the logarithmic boundaries naturally prevent \(p_j\) from hitting zero, these boundary constraints do not bind, allowing us to focus entirely on the primary equality multiplier \(\lambda\)).

    15.7 Closed-Form Derivation of the Empirical Frequency Solution

    Because the Lagrangian function is an unconstrained landscape, we can locate its optimal peak by computing the partial derivative with respect to an individual class parameter \(p_j\) and setting the vector result to zero:

    \[\frac{\partial \mathcal{L}_{\text{Lag}}}{\partial p_j} = \frac{\partial}{\partial p_j} \left[ \frac{1}{N} \sum_{i=1}^{N} z_{i,j} \ln p_j \right] - \frac{\partial}{\partial p_j} \left[ \lambda \sum_{j=1}^{M} p_j \right] = 0\]

    Using standard scalar derivative calculus (\(\frac{d}{dx} \ln x = \frac{1}{x}\)), this evaluates to:

    \[\frac{1}{N \cdot p_j} \sum_{i=1}^{N} z_{i,j} - \lambda = 0\]

    Let us isolate the parameter \(p_j\) in terms of our uncalculated multiplier \(\lambda\):

    \[\frac{1}{N \cdot p_j} \sum_{i=1}^{N} z_{i,j} = \lambda \implies p_j = \frac{1}{\lambda \cdot N} \sum_{i=1}^{N} z_{i,j}\]

    To solve for the value of \(\lambda\), we sum this entire equation across all \(M\) categories simultaneously:

    \[\sum_{j=1}^{M} p_j = \sum_{j=1}^{M} \left[ \frac{1}{\lambda \cdot N} \sum_{i=1}^{N} z_{i,j} \right]\]

    Invoking our structural sum-to-one constraint (\(\sum_{j=1}^M p_j = 1\)), we rewrite the left side of the equation:

    \[1 = \frac{1}{\lambda \cdot N} \sum_{i=1}^{N} \left[ \sum_{j=1}^{M} z_{i,j} \right]\]

    As established in Section 3, because every data row \(\vec{z}_i\) is a valid one-hot vector, the internal summation over \(j\) is always exactly \(1\):

    \[1 = \frac{1}{\lambda \cdot N} \sum_{i=1}^{N} [1]\]

    Summing the constant scalar \(1\) across all \(N\) data rows simply returns \(N\):

    \[1 = \frac{1}{\lambda \cdot N} \cdot N \implies 1 = \frac{1}{\lambda} \implies \lambda = 1\]

    We have successfully resolved the Lagrange multiplier (\(\lambda = 1\)). We substitute this value back into our parameter equation to extract our final, closed-form Maximum Likelihood Estimator:

    \[p_j^* = \frac{1}{1 \cdot N} \sum_{i=1}^{N} z_{i,j} \implies p_j^* = \frac{\sum_{i=1}^{N} z_{i,j}}{N}\]

    The Final Realization

    Look at that final equation. The numerator \(\sum_{i=1}^N z_{i,j}\) simply counts the total number of times the category \(a_j\) appeared in your training dataset folder. The denominator \(N\) is the total number of samples collected.

    This completes the derivation. Calculating a simple relative empirical frequency is the mathematically exact method for minimizing the Kullback-Leibler divergence over a discrete Multinoulli support manifold.

    Textbook Notes Study Guide

    The Generalist's Mathematical Blueprint: 1. The One-Hot Vector Transformation (\(\vec{z}_i\)) converts non-numeric categorical data into structural binary coordinates, providing a clean algebraic framework for matrix calculus. 2. The Exponent Product Trick (\(\prod p_j^{z_{i,j}}\)) acts as an elegant mathematical switch to express discrete categorical densities in a single differentiable line. 3. The Constrained Landscape Requirement: When maximizing probability parameters, unconstrained derivatives fail. We must use Lagrange Multipliers to build a bounded Lagrangian manifold (\(\mathcal{L}_{\text{Lag}}\)) ensuring the total volume conserves to exactly \(1.0\).

    The professor closes the lecture by pointing out a crucial detail. We now know how to calculate closed-form maximum likelihood solutions for single continuous populations (Multivariate Gaussians) and single discrete populations (Multinoulli Categoricals).

    However, in real-world machine learning deployments, data landscapes contain multiple, overlapping sub-populations mixed together. This means our log-likelihood equations lose their neat, convex properties, making them impossible to solve with simple closed-form derivatives.

    In the upcoming lectures, we will officially introduce advanced Estimation Theory, exploring how to fit complex, multi-peaked data distributions by running the iterative Expectation-Maximization (EM) Algorithm.
