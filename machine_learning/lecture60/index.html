    Mathematical Foundations of Machine Learning

    Chapter 45: Mixture Densities, Topological Quantization, and the Limiting Derivation of Hard K-Means from Isotropic GMMs

    45.1 The Latent Continuum: Posteriors as Structural Maps

    In Chapter 44, we completed our rigorous exploration of non-parametric ensembles, unrolling the exact functional calculus behind gradient-driven boosting networks. We now return to Unsupervised Topologies, expanding the latent variable frameworks introduced in Chapter 43 into concrete, deployable coordinate partition systems.

    Recall our core definition of an unsupervised learning engine: it is the systematic estimation of the global marginal distribution \(P(X)\) over an observed training dataset \(D\), bypassing the need for an external target label variable \(Y\). To capture complex, multi-modal data manifolds, we factor our model marginal distribution through a hidden Latent Variable space (\(\mathbb{Z}\)):

    \[P_{\vec{\theta}}(\vec{x}) = \int_{\mathbb{Z}} P_{\vec{\theta}}(\vec{x}, \,\, \vec{z}) \, d\vec{z} \quad \sim \quad P_{\vec{\theta}}(\vec{x}) = \sum_{\vec{z} \in \mathbb{Z}} P_{\vec{\theta}}(\vec{x}, \,\, \vec{z})\]

    Once this global marginal identity is optimized via Empirical Risk Minimization (ERM), we exploit its mathematical structure by deriving the Bayesian Posterior Distribution \(P_{\vec{\theta}}(Z \mid X)\):

    \[P_{\vec{\theta}}(\vec{z} \mid \vec{x}) = \frac{P_{\vec{\theta}}(\vec{x} \mid \vec{z})P(\vec{z})}{P_{\vec{\theta}}(\vec{x})}\]

    The categorical blueprint of our unsupervised algorithm depends entirely on the topological definition of this hidden space:

    • Continuous Latent Spaces (\(\mathbb{Z} \in \mathbb{R}^M\)): Computing the posterior mode maps coordinates into the domain of Continuous Representation Learning, Feature Extraction, or Embedding Generation. This path preserves continuous distance metrics across low-dimensional bottleneck sub-manifolds.
    • Discrete Latent Spaces (\(\mathbb{Z} \in \{1, \,\, 2, \,\, \dots, \,\, M\}\)): Computing the posterior distribution maps coordinates directly into a Clustering Framework. The hidden states act as discrete partition zones that segment the data space.

    45.2 Soft Topological Quantization: Gaussian Mixture Models (GMM)

    Let us unroll the exact mathematical physics of a discrete latent variable model configured as a Gaussian Mixture Model (GMM). We define a discrete latent variable \(Z\) capable of plucking one out of \(M\) total constituent groups, where each group index is governed by a categorical prior parameter \(\alpha_m\):

    \[P(z = m) = \alpha_m \quad \text{such that} \quad \sum_{m=1}^{M} \alpha_m = 1.0 \quad \text{and} \quad \alpha_m \geq 0\]

    Conditional on selecting a specific hidden component index \(m\), the raw data coordinates are modeled as manifesting from an unconstrained multivariate Gaussian distribution characterized by a localized mean vector \(\vec{\mu}_m\) and a covariance matrix \(\mathbf{\Sigma}_m\):

    \[P_{\vec{\theta}}(\vec{x} \mid z = m) = \mathcal{N}\left(\vec{x}; \,\, \vec{\mu}_m, \,\, \mathbf{\Sigma}_m\right) = \frac{1}{(2\pi)^{D/2} |\mathbf{\Sigma}_m|^{1/2}} \exp\left( -\frac{1}{2} (\vec{x} - \vec{\mu}_m)^T \mathbf{\Sigma}_m^{-1} \vec{x} - \vec{\mu}_m) \right)\]

    Precise Variable Typing Ledger:

    • \(\vec{x} \in \mathbb{R}^{D \times 1}\): The observed high-dimensional data vector containing \(D\) continuous features.
    • \(M \in \mathbb{N}\): The structural hyperparameter tracking the total count of allowed constituent clusters.
    • \(\vec{\mu}_m \in \mathbb{R}^{D \times 1}\): The geometric centroid vector of the \(m\)-th Gaussian component density.
    • \(\mathbf{\Sigma}_m \in \mathbb{R}^{D \times D}\): The full positive-definite covariance matrix controlling the spatial orientation and multi-dimensional spread of component \(m\).
    • \(\vec{\theta} = \{ \alpha_1 \dots \alpha_M, \,\, \vec{\mu}_1 \dots \vec{\mu}_M, \,\, \mathbf{\Sigma}_1 \dots \mathbf{\Sigma}_M \}\): The unified global parameter ledger.

    We expand our global marginal density by summing across this discrete component bank:

    \[P_{\vec{\theta}}(\vec{x}) = \sum_{m=1}^{M} \alpha_m \mathcal{N}\left(\vec{x}; \,\, \vec{\mu}_m, \,\, \mathbf{\Sigma}_m\right)\]

    The Expectation-Maximization (EM) Sub-Routine

    Because our data matrix \(\mathbf{X}\) lacks explicit label assignments pinpointing which specific cluster generated each row, we cannot solve for \(\vec{\theta}\) using simple closed-form maximum likelihood equations. We optimize the parameter space iteratively using the Expectation-Maximization (EM) Algorithm.

    During each execution loop, the engine alternates between calculating fractional hidden coordinate distributions (the E-Step responsibilities) and maximizing our global Expected Lower Bound (ELBO) matrix to update our structural parameter positions (the M-Step):

    \[\text{E-Step (Responsibility Mapping):} \quad \gamma_{im} = P_{\vec{\theta}}(z = m \mid \vec{x}_i) = \frac{\alpha_m \mathcal{N}\left(\vec{x}_i; \,\, \vec{\mu}_m, \,\, \mathbf{\Sigma}_m\right)}{\sum_{j=1}^{M} \alpha_j \mathcal{N}\left(\vec{x}_i; \,\, \vec{\mu}_j, \,\, \mathbf{\Sigma}_j\right)}\]
    \[\text{M-Step (Centroid Realignment):} \quad \vec{\mu}_m^{\text{\,\,new}} = \frac{\sum_{i=1}^n \gamma_{im} \vec{x}_i}{\sum_{i=1}^n \gamma_{im}}\]

    The Soft Clustering Footprint

    Once these parameters settle, feeding an un-labeled test coordinate \(\vec{x}\) into our posterior engine outputs an \(M\)-dimensional categorical distribution vector. We classify this as Soft Clustering.

    Because a standard Gaussian distribution possesses infinite mathematical support across Euclidean space (\(\mathbb{R}^D\)), every component density overlaps. Consequently, the posterior vector assigns a non-zero fractional probability to every single cluster slot simultaneously. The model tracks a continuous gradient of regional responsibility, declaring that a sample belongs to cluster 1 with \(70\%\) probability and cluster 2 with \(30\%\) probability.

    45.3 Hard Topological Quantization: The Isotropic Dirac Collapse

    While tracking continuous soft responsibilities provides exceptional probabilistic expressivity, practical software architectures often require a definitive, discrete coordinate allocation. We define this paradigm as Hard Clustering. Under this setup, a sample must be locked completely to a single isolated cluster category, forcing the posterior distribution to collapse into a rigid Dirac Delta Distribution:

    \[P_{\vec{\theta}}(z = m \mid \vec{x}_i) \in \{0, \,\, 1\} \quad \text{such that} \quad \gamma_{im} = 1.0 \quad \text{for exactly one slot } m^*\]

    To force a standard GMM to natively generate this hard, non-overlapping spatial partition by construction, we must enforce an aggressive Isotropic Structural Prior constraint on our component covariance matrices.

    Let us lock every component covariance matrix to a uniform, spherical identity structure scaled by a single, shared scalar variance parameter \(\sigma^2\):

    \[\mathbf{\Sigma}_m = \sigma^2 \mathbf{I} \quad \forall \quad m \in \{1, \,\, 2, \,\, \dots, \,\, M\}\]

    We unroll our structural component equation under this isotropic identity constraint:

    \[\mathcal{N}\left(\vec{x}_i; \,\, \vec{\mu}_m, \,\, \sigma^2 \mathbf{I}\right) = \frac{1}{(2\pi\sigma^2)^{D/2}} \exp\left( -\frac{\|\vec{x}_i - \vec{\mu}_m\|_2^2}{2\sigma^2} \right)\]

    We substitute this isotropic equation directly back into our E-step posterior responsibility mapping formula to evaluate how individual coordinates behave:

    \[\gamma_{im} = \frac{\alpha_m \frac{1}{(2\pi\sigma^2)^{D/2}} \exp\left( -\frac{\|\vec{x}_i - \vec{\mu}_m\|_2^2}{2\sigma^2} \right)}{\sum_{j=1}^{M} \alpha_j \frac{1}{(2\pi\sigma^2)^{D/2}} \exp\left( -\frac{\|\vec{x}_i - \vec{\mu}_j\|_2^2}{2\sigma^2} \right)}\]

    We cancel out the shared multi-dimensional scale denominators operating uniformly across both the numerator and the denominator:

    \[\gamma_{im} = \frac{\alpha_m \exp\left( -\frac{\|\vec{x}_i - \vec{\mu}_m\|_2^2}{2\sigma^2} \right)}{\sum_{j=1}^{M} \alpha_j \exp\left( -\frac{\|\vec{x}_i - \vec{\mu}_j\|_2^2}{2\sigma^2} \right)}\]

    Now, we analyze the limiting behavior of this coordinate map as we force our shared variance parameter to shrink down to an infinitesimally small, near-zero boundary (\(\sigma^2 \to 0\)):

    \[\lim_{\sigma^2 \to 0} \gamma_{im} = \lim_{\sigma^2 \to 0} \frac{\alpha_m \exp\left( -\frac{\|\vec{x}_i - \vec{\mu}_m\|_2^2}{2\sigma^2} \right)}{\sum_{j=1}^{M} \alpha_j \exp\left( -\frac{\|\vec{x}_i - \vec{\mu}_j\|_2^2}{2\sigma^2} \right)}\]

    Look closely at the mathematics of this limit. The terms inside the exponentials are driven by negative quotients scaled by \(\frac{1}{\sigma^2}\). As \(\sigma^2\) vanishes, the exponent components explode toward negative infinity, forcing the overall exponential terms to race toward zero.

    However, the velocity of this decay is governed entirely by the numerator term: the squared Euclidean Distance (\(\|\vec{x}_i - \vec{\mu}\|_2^2\)).

    Let \(m^*\) denote the specific component index that sits physically closest to our data point coordinate \(\vec{x}_i\) in our vector space:

    \[m^* = \arg\min_{j \in \{1 \dots M\}} \|\vec{x}_i - \vec{\mu}_j\|_2^2\]

    If we divide both the numerator and denominator by the dominant exponential term associated with this closest centroid (\(\exp( -\|\vec{x}_i - \vec{\mu}_{m^*}\|_2^2 / 2\sigma^2 )\)), all neighboring component ratios collapse straight to absolute zero because their larger spatial distances force an infinitely faster exponential decay. This isolates a clean binary limit statement:

    \[\lim_{\sigma^2 \to 0} \gamma_{im} = \begin{cases} 1.0 & \text{if } m = \arg\min_j \|\vec{x}_i - \vec{\mu}_j\|_2^2 \\ 0.0 & \text{otherwise} \end{cases}\]

    This completes our foundational derivation. By locking our mixture model components to isotropic structures and forcing the variance to zero, the continuous soft responsibilities of our EM algorithm undergo an absolute phase transition. The overlapping densities collapse into rigid boundaries, giving rise to the classic hard assignment rules of the K-Means Clustering Algorithm.

    45.4 The Derived K-Means Algorithmic Protocol

    By unrolling this isotropic limit case, the standard loops of the EM algorithm simplify into the two alternating steps of the non-parametric K-Means pipeline.

    Step 1: The Expectation Step (Hard Voronoi Assignment)

    The continuous responsibility matrix collapses into an absolute binary assignment gate. The engine loops across all \(n\) training data rows, calculating the exact Euclidean distance from each sample to all \(M\) current cluster centroids. It assigns each observation exclusively to its closest centroid set \(S_m\):

    \[S_m = \left\{ \vec{x}_i \in D \,\,\Big|\,\, m = \arg\min_{j \in \{1 \dots M\}} \|\vec{x}_i - \vec{\mu}_j\|_2^2 \right\}\]

    This step draws a system of hard geometric boundaries across the data manifold, partitioning the feature space into non-overlapping cells known as a Voronoi Tessellation.

    Step 2: The Maximization Step (Arithmetic Centroid Realignment)

    Once every sample is mapped to a discrete cell, the parameter optimization step runs. Because responsibilities are strictly binary (\(\gamma_{im} \in \{0, 1\}\)), the soft GMM matrix equations collapse into a clean Arithmetic Sample Mean calculation.

    The position of each cluster centroid \(\vec{\mu}_m\) is updated by averaging the coordinates of all training observations that were routed into its Voronoi cell during the assignment phase:

    \[\vec{\mu}_m = \frac{1}{|S_m|} \sum_{\vec{x}_i \in S_m} \vec{x}_i\]

    45.5 Non-Linear Manifolds: The Kernelized K-Means Extension

    Because the standard K-Means algorithm evaluates spatial proximity using raw Euclidean distances (\(\|\vec{x}_i - \vec{\mu}_j\|_2^2\)), it possesses a hard geometric limitation: it can only isolate linearly separable, spherical cluster volumes. If a dataset is distributed along a complex non-linear manifold—such as two concentric rings or entangled sinusoidal waves—K-Means fails completely, cutting straight across the structural paths.

    To bypass this linear boundary without abandoning our fast quantization algorithm, we deploy Kernelized K-Means. We map our raw features into an expanded Hilbert feature space \(Z\) using a non-linear coordinate transformation mapping function \(\vec{\phi}(\vec{x})\):

    \[\vec{\phi}(\vec{x}): \mathbb{R}^D \to Z\]

    Let us evaluate our assignment distance equation within this high-dimensional feature domain:

    \[\|\vec{\phi}(\vec{x}_i) - \vec{\mu}_m^{\,\,\phi}\|_2^2 = \vec{\phi}(\vec{x}_i) \cdot \vec{\phi}(\vec{x}_i) - 2 \vec{\phi}(\vec{x}_i) \cdot \vec{\mu}_m^{\,\,\phi} + \vec{\mu}_m^{\,\,\phi} \cdot \vec{\mu}_m^{\,\,\phi}\]

    We substitute our M-step arithmetic centroid update equation (\(\vec{\mu}_m^{\,\,\phi} = \frac{1}{|S_m|} \sum_{\vec{x}_k \in S_m} \vec{\phi}(\vec{x}_k)\)) directly into this inner product expansion:

    \[\|\vec{\phi}(\vec{x}_i) - \vec{\mu}_m^{\,\,\phi}\|_2^2 = \vec{\phi}(\vec{x}_i) \cdot \vec{\phi}(\vec{x}_i) - \frac{2}{|S_m|} \sum_{\vec{x}_k \in S_m} \vec{\phi}(\vec{x}_i) \cdot \vec{\phi}(\vec{x}_k) + \frac{1}{|S_m|^2} \sum_{\vec{x}_k \in S_m} \sum_{\vec{x}_l \in S_m} \vec{\phi}(\vec{x}_k) \cdot \vec{\phi}(\vec{x}_l)\]

    Look at the structure of this expanded equation. Every single vector transformation has assembled into a clean dot product pair (\(\vec{\phi}(\cdot) \cdot \vec{\phi}(\cdot)\)). This means we can deploy the Kernel Trick derived in Chapter 33, replacing every high-dimensional inner product with a localized evaluation of a Mercer Kernel function \(K(\vec{x}_i, \vec{x}_j)\):

    \[\|\vec{\phi}(\vec{x}_i) - \vec{\mu}_m^{\,\,\phi}\|_2^2 = K(\vec{x}_i, \,\, \vec{x}_i) - \frac{2}{|S_m|} \sum_{\vec{x}_k \in S_m} K(\vec{x}_i, \,\, \vec{x}_k) + \frac{1}{|S_m|^2} \sum_{\vec{x}_k \in S_m} \sum_{\vec{x}_l \in S_m} K(\vec{x}_k, \,\, \vec{x}_l)\]

    This is a beautiful result. It proves that we can execute K-Means clustering over immensely complex, non-linear manifolds without ever calculating the explicit high-dimensional coordinate mapping \(\vec{\phi}(\vec{x})\) directly. As long as we can define a valid kernel matrix over our data rows, the engine can compute exact hard assignments and track non-linear clusters flawlessly.

    45.6 The Semantics of Representation Bias: Supervised vs. Unsupervised Partitions

    To conclude this chapter, we analyze the core difference between Implicit Latent Variable Models built via supervised networks versus those optimized via unsupervised marginal goals.

    Every layer inside a deep neural network functions as a coordinate projection engine that shapes an internal hidden representation space \(\vec{z}\). However, if you train a deep network (like a multi-layer MLP) on a supervised task, the learned representation coordinates are heavily constrained by the specific semantics of the training labels.

    [Image contrasting supervised classification partitions vs unsupervised marginal clustering geometry]

    The MNIST Failure Mode Analysis

    To visualize this limitation, consider the standard MNIST handwritten digit dataset. Suppose you train a deep classifier to predict the raw digit category (\(Y \in \{0 \dots 9\}\)).

    • The Supervised Representation Landscape: The network optimizes its early layers to isolate features that separate digit identity (e.g., detecting the closed loop that isolates a 0 from a 1). During this process, it deliberately erases any feature variation that is irrelevant to the target labels—such as the image's rotation, stroke thickness, or line orientation.

    If you freeze this model, tap its intermediate hidden layers to extract embeddings, and try to use those features to solve a new downstream task—such as classifying the angle of orientation of the writing—the model fails completely. The features required to track orientation were discarded as non-target noise during the initial supervised training loop.

    The Unsupervised Marginal Defense

    Unsupervised latent variable models completely avoid this representation bias. Because an unsupervised architecture is optimized to estimate the absolute global marginal distribution \(P(X)\), the model is forced to preserve the entire structural topology of the data manifold. By mapping the data coordinates without relying on a pre-selected labeling scheme, the resulting continuous embeddings (\(\vec{z}\)) capture a comprehensive structural summary of the input features. This makes unsupervised representations highly robust and versatile, allowing them to easily scale and transfer to support diverse, unannounced downstream tasks in production environments.
