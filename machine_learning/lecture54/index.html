    Mathematical Foundations of Machine Learning

    Chapter 43: Regression Trees, Continuous Threshold Search Heuristics, and the Mechanics of Variance-Reduction Ensembles (Bagging)

    43.1 The Transition to Continuous Target Metrics: Regression Trees

    In Chapter 42, we unrolled the mathematical formulation of Classification and Regression Trees (CART), focusing on greedy recursive partitioning over high-dimensional discrete probability distributions. Under that classification blueprint, regional purity was evaluated using information-theoretic metrics—such as Shannon Entropy or Gini Impurity—that required counting class frequencies.

    We now transition to the secondary task family of non-parametric architectures: Regression Trees. When the global target variable \(Y\) tracks a continuous scalar space (\(Y \in \mathbb{R}\)), discrete probability frequency metrics collapse completely. We cannot calculate the entropy of a continuous, infinite real-valued manifold using discrete counters.

    To resolve this metric deficit, we must alter the operational assignment rules running inside our partitioned regions. Suppose our top-down greedy search splits an active parent node \(m\) into a regional boundary block denoted as \(R_m\). This sub-manifold currently encompasses a subset of \(N_m\) training observations.

    Because we can no longer assign a categorical majority label to this zone, we regularize our regional prediction by calculating a flat, localized summary value. We define the continuous target prediction for any input coordinate landing within the spatial domain of region \(R_m\) as the Maximum Likelihood Arithmetic Mean (\(\hat{y}_m\)) of all target labels currently trapped within that zone:

    \[\hat{y}_m = \frac{1}{N_m} \sum_{\vec{x}_i \in R_m} y_i\]

    During downstream inference, when an unseen testing sample vector \(\vec{x}_{\text{test}}\) executes its deterministic traversal pass down the tree graph, it travels through axis-parallel gates until it lands in a terminal Leaf Node. The model ignores the individual variations of the testing coordinate and assigns it a single, flat prediction value matching the pre-calculated arithmetic mean \(\hat{y}_{\text{leaf}}\) of that terminal region.

    The Continuous Probability Horizon

    A student noted an elegant architectural parallel: just as a classification leaf node outputs a highly skewed probability distribution over discrete slots, a properly optimized regression leaf node outputs a tightly clustered distribution over the continuous real numbers. When a tree is grown correctly, each sequential split maximizes localized density concentration. Rather than generating flat, uniform chaos across the target line, the recursive cuts isolate uniform clusters, forcing the targets trapped in a leaf zone to gather closely around their regional mean.

    43.2 The Continuous Variance Impurity Metric

    To guide our top-down greedy splitting optimizer when target labels are continuous, we must swap out our information-theoretic impurity functions for a metric calibrated to real-valued distributions. We model the impurity of a continuous region \(R_m\) by evaluating its Internal Structural Variance (Mean Squared Error):

    \[I(R_m) = \frac{1}{N_m} \sum_{\vec{x}_i \in R_m} \left( y_i - \hat{y}_m \right)^2\]

    This formulation acts as a direct evaluation of splitting quality. If a split forces a wide variety of wildly differing target numbers to crowd together into a single region, the arithmetic mean \(\hat{y}_m\) will function as a poor summary vector. The squared differences will expand, flagging the region as highly impure.

    To execute a split at node \(m\), our greedy optimization engine evaluates a candidate feature index \(j\) and a continuous axis threshold \(\tau\), minimizing the combined weighted variance of the resulting left child (\(R_L\)) and right child (\(R_R\)) sub-regions:

    \[\mathcal{L}(j, \,\, \tau) = \frac{N_L}{N_m} \left[ \frac{1}{N_L} \sum_{\vec{x}_i \in R_L} (y_i - \hat{y}_L)^2 \right] + \frac{N_R}{N_m} \left[ \frac{1}{N_R} \sum_{\vec{x}_i \in R_R} (y_i - \hat{y}_R)^2 \right]\]

    Canceling out the fractional headcounts unrolls a clean, total sum-of-squares objective function:

    \[\mathcal{L}(j, \,\, \tau) = \frac{1}{N_m} \left[ \sum_{\vec{x}_i \in R_L(j, \tau)} (y_i - \hat{y}_L)^2 + \sum_{\vec{x}_i \in R_R(j, \tau)} (y_i - \hat{y}_R)^2 \right]\]

    The optimizer's core objective is to discover a splitting boundary that maximizes the statistical distance between the means of the child nodes (\(\hat{y}_L\) and \(\hat{y}_R\)). By segregating smaller numbers to the left and larger numbers to the right, the internal variance within each child node drops to a minimum, generating tight, highly pure localized prediction blocks.

    43.3 Algorithmic Unrolling of Continuous Threshold Selection

    When a chosen feature dimension \(X_j\) is a continuous variable mapping to an infinite set of real numbers (\(\mathbb{R}\)), an engineer faces a massive practical hurdle: how can a software script scan an infinite number of candidate thresholds \(\tau\) to find the mathematical minimum of our variance ledger? To implement this on real digital hardware, we convert the infinite continuous space into a discrete set of candidate boundaries using a rigorous Rank-Ordering Midpoint Heuristic.

    The Midpoint Search Protocol

    Suppose an active node \(m\) contains \(N_m\) training samples. To calculate all valid thresholds for a specific continuous feature axis \(X_j\), the execution engine runs the following algorithmic pipeline:

    1. Feature Extraction: Extract the \(N_m\) scalar coefficients for feature \(X_j\) from all training observations trapped within the active node domain.
    2. Sort and De-duplicate: Run a sorting pass over these scalar entries, strip out any duplicate values, and arrange the remaining unique numbers into a strictly increasing, rank-ordered chronological sequence:
    \[U_1 < U_2 < U_3 < \dots < U_P \quad \text{where} \quad P \leq N_m\]
    1. Midpoint Generation: Define the finite set of candidate splitting thresholds \(\tau_k\) by calculating the exact arithmetic midpoints between adjacent rank-ordered values:
    \[\tau_k = \frac{U_k + U_{k+1}}{2} \quad \forall \quad k \in \{1, \,\, 2, \,\, \dots, \,\, P-1\}\]

    The Algorithmic Complexity Gap: \(\mathcal{O}(N^2)\) vs. \(\mathcal{O}(N \log N)\)

    The professor notes a vital computational bottleneck. If you implement this search naively—sorting the array, picking a threshold, and then running a full data scan over all \(N_m\) rows to compute the child means and variance losses for each candidate midpoint—the algorithmic complexity scales quadratically as \(\mathcal{O}(N_m^2)\) per feature. When processing millions of samples, this quadratic scaling paralyzes the training engine.

    To accelerate this step, modern optimization frameworks optimize the execution complexity down to \(\mathcal{O}(N_m \log N_m)\) using a Single-Pass Running Sum Scan.

    Instead of recalculating the child averages (\(\hat{y}_L, \hat{y}_R\)) from scratch at each candidate midpoint, the engine initializes a global tracking buffer that computes the total sum and squared sum of all target variables in the parent node.

    Because the feature values are pre-sorted, stepping from threshold \(\tau_k\) to \(\tau_{k+1}\) shifts exactly one data observation from the right child node into the left child node. The engine dynamically updates the child sum statistics using basic, constant-time scalar arithmetic:

    \[\text{Sum}(R_L)^{\text{new}} = \text{Sum}(R_L)^{\text{old}} + y_{\text{shifted}}\]
    \[\text{Sum}(R_R)^{\text{new}} = \text{Sum}(R_R)^{\text{old}} - y_{\text{shifted}}\]

    By tracking running sums, the child means and variance criteria are evaluated in a single linear pass (\(\mathcal{O}(N_m)\)) after the initial sorting step (\(\mathcal{O}(N_m \log N_m)\)), dropping total layer compute down to an efficient \(\mathcal{O}(N \log N)\) footprint.

    Industrial Acceleration: The Histogram Method

    In modern, high-velocity tree implementations (such as LightGBM or XGBoost Sketched engines), sorting raw floating-point numbers is completely abandoned.

    The algorithm passes the continuous feature values into a discrete 1D Histogram Filter, grouping the inputs into a fixed number of bins (e.g., 256 bins). The system discards the raw coordinates entirely, using the hard physical edges of the histogram bins as its sole candidate thresholds. This quantization step drops training memory overhead and converts the threshold search into a blindingly fast linear calculation.

    43.4 Regularization Frameworks: Pruning Rules and Stopping Criteria

    If we run our recursive greedy splitting algorithm without constraints, the tree graph will continue growing until it maps every single unique training sample into its own dedicated leaf node.

    In this unconstrained state, the network transforms into a rigid Memory Lookup Table. It achieves absolute zero training error (\(\hat{R} = 0\)), but its spatial regions shrink to microscopic volumes.

    This condition represents the ultimate manifestation of High Variance / Low Bias Overfitting. The tree perfectly memorizes the background noise of the training coordinates, but fails completely when evaluated on unseen validation data. To control this variance and force the model into a stable, low-variance regime, we must deploy strict Architectural Regularizers to stop tree growth.

      [ Unregularized Deep Tree ]               [ Regularized Shallow Tree ]
  - Region Volume: Near Zero (Microscopic)  - Region Volume: Large Hyper-Rectangles
  - Variance: Extremely High (Overfit)      - Variance: Suppressed and Controlled
  - Bias: Absolute Zero                     - Bias: Higher (Structured Smoothness)
  - Profile: Explicit Lookup Table          - Profile: Generalized Continuous Predictor

    We constrain the tree's hypothesis class by enforcing four classical hyperparameter constraints:

    1. Maximum Depth Limit (\(D_{\max}\))

    We define a hard ceiling on the maximum depth of the tree graph. If an active node sits at a depth coordinate equal to \(D_{\max}\), the optimization engine freezes the node instantly, blocking any further cuts. This parameter enforces a minimum geometric volume threshold on the partitioned hyper-rectangles.

    2. Minimum Spatial Headcount (\(N_{\min}\))

    The engine checks the total headcount of observations trapped inside an active node domain (\(N_m\)). If \(N_m\) falls below a pre-selected integer threshold \(N_{\min}\), the node is declared a terminal leaf, and splitting halts. This parameter operates exactly like the window constraints utilized in non-parametric density estimation frameworks.

    3. Minimum Child Allocation

    A split is physically blocked if the proposed boundary threshold \(\tau\) routes a tiny cluster of samples into either child node (\(N_L < N_{\text{leaf\_min}}\) or \(N_R < N_{\text{leaf\_min}}\)). This prevents the optimizer from carving off isolated outliers to cheat the variance ledger.

    4. Minimum Impurity Improvement (\(\epsilon\))

    The engine calculates the net reduction in impurity achieved by a candidate cut:

    \[\Delta I = I(R_m) - \left[ \frac{N_L}{N_m}I(R_L) + \frac{N_R}{N_m}I(R_R) \right]\]

    The split is executed only if this gain exceeds a positive hyperparameter threshold (\(\Delta I \geq \epsilon\)). If the variance reduction is near-zero, the cut is discarded as statistically insignificant noise, protecting the layer from over-partitioning.

    43.5 Native Categorical Axis Processing

    A massive operational advantage of tree-based models over parametric neural networks is their ability to process Categorical Features natively, without requiring any preprocessing transformations like one-hot encoding.

    Suppose a data dimension tracking feature \(X_j\) represents discrete, descriptive color categories: \(X_j \in \{\text{Red}, \,\, \text{Green}, \,\, \text{Blue}\}\). If you feed this categorical field into a neural network, you must convert it into a one-hot matrix representation (\([1,0,0], [0,1,0]\)).

    This conversion scales up the input dimensionality, creating sparse matrices that dilute the gradient signals during backpropagation.

    Decision trees bypass this matrix degradation completely by altering the mathematical nature of their splitting predicates. Instead of running continuous directional inequalities (\(X_j \leq \tau\)), a categorical tree node executes an explicit Set Membership Predicate:

    \[\vec{x} \in R_L \quad \text{if} \quad x_j \in \mathcal{S}\]
    \[\vec{x} \in R_R \quad \text{if} \quad x_j \notin \mathcal{S}\]

    Literal Variable Definition:

    • \(\mathcal{S}\): A discrete subset of categories carved out from the feature's total global pool.

    To optimize a categorical split, the greedy engine scans across different combinations of category groupings to find the specific partition subset \(\mathcal{S}\) that minimizes our standard weighted impurity ledger.

    Because the questions evaluate category membership directly (\(X_j \in \{\text{Red}, \text{Blue}\}\)), the system avoids numeric conversions entirely, preserving the structural integrity of discrete descriptive variables.

    43.6 Ensemble Theory: Combining Weak Learners to Compress Global Variance

    While regularizing single decision trees using depth limits or sample thresholds prevents extreme overfitting, it introduces a severe trade-off: highly regularized, shallow trees are weak function approximators. They possess high bias and lack the expressive capacity to model intricate, non-linear system boundaries effectively.

    This limitation led to the development of Ensemble Learning. Instead of searching for a single, perfect master hypothesis, ensemble frameworks combine a large committee of easily learnable, low-capacity models—formally known as Weak Learners—to build a highly robust global predictor.

    To see why this approach works, let us look back at our foundational true risk decomposition:

    \[\text{True Risk Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}\]

    Recall that the statistical variance component measures how violently our chosen hypothesis function swings across parameter space when it is trained on entirely different mini-batches of data pulled from the global distribution \(P(X,Y)\):

    \[\text{Variance} = \mathbb{E}_D \left[ \left( h_D(\vec{x}) - \bar{h}(\vec{x}) \right)^2 \right]\]

    The Theoretical Ideal: The Ensemble Average

    Suppose we have access to a magical reservoir of \(B\) completely independent datasets pulled from the environment. We train a separate decision tree \(h_{D_b}(\vec{x})\) on each independent dataset. If we construct a global ensemble predictor \(h_{\text{ens}}(\vec{x})\) by averaging the predictions of all these independent trees, the true mathematical mean remains anchored:

    \[\bar{h}(\vec{x}) = \mathbb{E}_D\big[h_D(\vec{x})\Big]\]

    Let us evaluate the variance profile of this averaged ensemble model:

    \[\text{Var}\left( h_{\text{ens}}(\vec{x}) \right) = \text{Var}\left( \frac{1}{B}\sum_{b=1}^{B} h_{D_b}(\vec{x}) \right)\]

    Because each dataset was sampled completely independently, the individual models are entirely uncorrelated. From probability theory, the variance of the sum of independent random variables scales inversely with the number of samples:

    \[\text{Var}\left( h_{\text{ens}}(\vec{x}) \right) = \frac{1}{B^2} \sum_{b=1}^{B} \text{Var}\left( h_{D_b}(\vec{x}) \right) = \frac{1}{B^2} \cdot B \cdot \sigma_{\text{single}}^2 = \frac{\sigma_{\text{single}}^2}{B}\]

    Look at the power of this algebraic result: as the number of independent estimators \(B\) scales up, the global variance of the ensemble decays toward absolute zero. This means you can configure each individual tree to be deeply partitioned and highly complex (low bias, high variance), and then eliminate that variance entirely simply by averaging them together, clearing a direct path to minimizing true risk.

    43.7 Bootstrap Aggregation (Bagging) over Correlated Manifolds

    In practical machine learning applications, the ideal ensemble scenario faces a major real-world hurdle: we do not possess an infinite reservoir of independent datasets. We have exactly one single dataset \(D\) containing a finite number of observations.

    If you train multiple deep decision trees on this exact same dataset, they will optimize into identical configurations. Their predictions will be perfectly correlated, completely neutralizing the variance-reduction benefits of averaging.

    To bypass this data constraint, we deploy an empirical sampling technique known as Bootstrap Aggregation, or Bagging.

                           ┌──> Resample with Replacement (Batch 1) ───> Train Tree 1 ──┐
                       │                                                            ▼
  [ Global Dataset D ] ─┼──> Resample with Replacement (Batch 2) ───> Train Tree 2 ──┼─> [ Average Prediction ]
                       │                                                            ▲
                       └──> Resample with Replacement (Batch B) ───> Train Tree B ──┘

    Instead of independent datasets, we generate \(B\) synthetic datasets by running a Random Bootstrap Sampling pass with Replacement over our single dataset \(D\).

    For each individual model head \(b\), we randomly pluck a sample from \(D\), record its coordinates, and drop it straight back into the global pool. We repeat this selection step exactly \(N\) times. Because samples are replaced after each pick, a single observation can be selected multiple times within a single bootstrap batch, while other observations are missed entirely.

    The Cost of Data Redundancy: The Correlation Penalty

    Because these bootstrap batches are pulled from the same underlying dataset \(D\), they violate our ideal independence assumption. The individual models are inherently correlated.

    Let us re-derive the variance profile of an ensemble average when its estimators share a positive correlation coefficient \(\rho\):

    \[\text{Var}\left( h_{\text{bag}}(\vec{x}) \right) = \rho \cdot \sigma^2 + \frac{1 - \rho}{B} \cdot \sigma^2\]

    Precise Coordinate Variables:

    • \(\sigma^2\): The native structural variance of a single unregularized decision tree.
    • \(B\): The total number of bagged tree estimators stacked inside the ensemble.
    • \(\rho\): The positive correlation coefficient measuring the dependency link between individual trees.

    Look closely at this formula as the ensemble size \(B\) scales toward infinity:

    \[\lim_{B \to \infty} \text{Var}\left( h_{\text{bag}}(\vec{x}) \right) = \rho \cdot \sigma^2\]

    The second term drops to zero, but the first term remains completely unaffected by the ensemble size. This reveals a strict mathematical ceiling: the variance reduction achieved by standard bagging is bottlenecked by the correlation coefficient \(\rho\) of its trees. If the trees are highly correlated, stacking more models adds no value. To drive the global variance down further, we must find an architectural trick to explicitly decorrelate the trees.

    43.8 Random Forests: Deep Decorrelation through Feature Subsampling

    This mathematical bottleneck brings us directly to one of the most powerful and widely used algorithms in machine learning: The Random Forest.

    Developed by Leo Breiman, a Random Forest is an advanced bootstrap aggregation ensemble that forces deep decorrelation across its constituent decision trees by adding an intentional restriction: Random Feature Subsampling.

    When building a standard bagged ensemble, each tree evaluates all \(D\) available feature dimensions at every single node split. Because dominant features (like a massive, high-variance target driver) will win the greedy cost optimization across almost all bootstrap batches, the resulting trees still look highly similar, locking the correlation coefficient \(\rho\) at a high level.

    A Random Forest breaks this correlation by changing the rules of the greedy search. As each individual tree grows, the algorithm explicitly blocks the node from scanning all \(D\) features. At every single node split, the engine randomly subsamples a tiny, isolated subset of \(m\) features (where an industry standard default is \(m \approx \sqrt{D}\)):

      Step 1: Bootstrap Resampling
  Generate B synthetic data batches from D via random selection with replacement.

  Step 2: Randomized Node-Level Feature Subsampling
  At each individual node split, lock out all features except a random subset of size m = √D.

  Step 3: Axis-Parallel Split Optimization
  Optimize the threshold exclusively over this restricted feature pool to minimize variance.

  Step 4: Continuous Multi-Tree Aggregation
  Compile the final ensemble prediction by taking the uniform average of all B tree heads.

    By forcing each node split to select from a highly restricted, randomly altered feature pool, the algorithm deliberately prevents dominant features from hijacking the tree structures. A feature that would normally dominate the optimization pass is locked out of most nodes, forcing the tree to discover alternative, hidden geometric structures across the manifold.

    This structural restriction drives down the correlation coefficient \(\rho\) between the trees. Returning to our limiting ensemble equation:

    \[\text{Variance Limit} = \rho \cdot \sigma^2\]

    By reducing the correlation penalty \(\rho\), Random Forests unlock the full mathematical power of ensemble averaging. They allow engineers to grow hundreds of deeply partitioned, low-bias decision trees in parallel, while using bootstrap aggregation and feature randomization to eliminate global variance, clearing a direct path to minimizing true risk on complex datasets.
